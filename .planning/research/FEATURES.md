# Feature Landscape

**Domain:** Korean stock information and screening web app (상한가 근접 스캐너 + AI 뉴스/토론 요약)
**Researched:** 2026-04-10
**Confidence:** MEDIUM — Korean-language sources directly checked; some app internals inferred from crawlers/reviews

---

## Table Stakes

Features users expect. Missing = product feels incomplete or untrustworthy.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| 실시간(또는 1분 폴링) 등락률 리스트 | 모든 HTS/MTS에서 제공. 없으면 정보 앱으로 인정받지 못함 | Low | 등락률 기준 내림차순 정렬, 빨강/파랑 색상 코드 필수 |
| 상한가 근접 종목 필터링 | 이 앱의 핵심 가치 명제 | Low–Med | 기준값(기본 25%) 사용자 조절. 코스피/코스닥 구분 표시 |
| 종목 현재가 + 등락률 + 거래량 표시 | 모든 증권 앱 기본 | Low | 전일 대비 등락폭(원) + 등락률(%) 동시 표시가 업계 표준 |
| 종목 검색 | 이름/코드 검색. 없으면 진입 불가 | Low | 자동완성(종목명 + 종목코드 동시 검색) |
| 종목별 뉴스 목록 | 네이버 금융, 다음 금융 모두 제공. 기대치가 됨 | Med | 헤드라인 + 출처 + 시간. 기사 원문 링크 |
| 네이버 종목토론방 글 표시 | 이 앱의 명시적 요구사항. 국내 개인 투자자 필수 정보 | Med | 글 제목 + 조회수 + 작성시간 목록 형태 |
| AI 뉴스 요약 | 이 앱의 핵심 차별화이자 요구사항. 2025년 에픽AI 등 경쟁자 등장으로 기대치 상승 | Med | Claude API 사용, 종목별 최근 뉴스 → 1–3줄 요약 |
| AI 토론방 요약 | 마찬가지로 명시적 요구사항 | Med | 종목토론방 최근 글 → 시장 심리(긍정/부정/중립) + 주요 이슈 |
| KOSPI / KOSDAQ 구분 표시 | 한국 투자자가 첫 번째로 보는 정보 | Low | 각 종목 옆에 시장 구분 배지 |
| 데이터 갱신 시각 표시 | 폴링 앱에서 신뢰성 표시를 위해 필수 | Low | "15:02 기준" 형태로 마지막 업데이트 시각 표시 |

## Differentiators

경쟁 앱 대비 차별화 포인트. 기대값은 아니지만 있으면 가치 있음.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| 조절 가능한 상한가 근접 임계값 | 트레이더마다 기준이 다름 (20%, 25%, 29% 등). 영웅문/HTS는 조건검색식 직접 작성 필요 — 이 앱은 슬라이더 한 번으로 설정 | Low | 기본값 25%, 10–29% 범위 |
| 뉴스 + 토론방 통합 요약 | 뉴스와 커뮤니티 감성을 한 화면에서. 현재 대부분 앱은 각각 별도 탭/화면 | Med | 두 소스를 합산한 "시장심리 종합" 뷰 |
| 급등 종목 실시간 랭킹 정렬 | 씽크풀의 "이시간 뜨거운 토론실", 알파스퀘어의 "급등주 순위"처럼 — 이 앱은 상한가 근접 + 거래량 급증 복합 정렬 | Med | 등락률 + 거래량 가중 스코어로 정렬 |
| 종목별 뉴스/토론 감성 지표 | 에픽AI가 "news sentiment" 수치화 제공. 이 앱은 Claude 요약 기반 긍/부정 레이블로 단순화 | Med–High | v1에서는 텍스트 레이블(긍정/부정/중립)로 단순하게 구현 가능 |
| 빠른 종목 전환 | 리스트에서 종목 클릭 시 즉시 뉴스+요약 로드. SPA 패턴으로 페이지 이동 없음 | Med | 리스트 + 상세 패널 분할 레이아웃 |

## Anti-Features

v1에서 의도적으로 만들지 않을 기능.

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| 포트폴리오/잔고 관리 | 증권사 API 연동 필요, OAuth 복잡도, 법적 책임 상승. 영웅문/나무 등이 이미 지배 | 관심종목(로컬스토리지)으로 대체, 실제 잔고는 증권사 앱 사용 유도 |
| 주문 실행(매수/매도) | 금융업 인허가 필요. v1 범위 완전 이탈 | 링크로 증권사 앱 이동 유도 |
| 차트 (캔들/기술 지표) | TradingView 임베드 없이 직접 구현 시 고비용. 키움/알파스퀘어가 이미 잘 함 | 현재가 숫자와 간단한 mini sparkline으로 충분 |
| 로그인/회원가입 | PROJECT.md에서 v2로 명시적 제외. 인증 없이 출시 | 브라우저 localStorage로 임시 상태 관리 |
| 관심종목 서버 저장 | 로그인 없으면 구현 불가 | localStorage로 클라이언트 저장, v2에서 sync |
| 푸시/알림 | Service Worker + 권한 UX 복잡도. 상한가 도달 알림은 오탐 리스크 높음 | 앱 내 시각적 강조(색상, 배지)로 대체 |
| AI 자동 매매 추천 | 법적 리스크 (투자자문업 간주 가능), 오탐 책임. 씽크풀도 면책 고지 필수 | 요약만 제공, "매수/매도" 언어 사용 금지 |
| 전체 시장 스크리너 (PER/PBR/ROE 필터) | 한국경제 스크리너, TradingView가 이미 잘 함. 차별화 없음 | 등락률 단일 지표에 집중 |
| 해외 주식 | PROJECT.md 명시적 제외 | v2 이후 |
| 모바일 앱(네이티브) | 웹 우선. 개발 비용 2배 | 모바일 반응형 웹으로 충분히 커버 |

## Feature Dependencies

```
종목 검색
  └→ 종목 상세 뷰 (뉴스 + 토론방 목록)
        └→ AI 뉴스 요약 (뉴스 수집 완료 후)
        └→ AI 토론방 요약 (토론방 수집 완료 후)

상한가 근접 스캔 리스트
  └→ 임계값 설정 UI (슬라이더)
  └→ 종목 클릭 → 종목 상세 뷰

데이터 수집 파이프라인
  ├→ 주식 시세 (1분 폴링 or WebSocket)
  ├→ 뉴스 수집 (네이버/한경/매경 크롤링 or API)
  └→ 네이버 종목토론방 수집 (크롤링)
```

**핵심 의존 관계:**
- AI 요약은 뉴스/토론방 수집이 먼저 구현되어야 가능
- 상한가 근접 리스트는 실시간 시세 파이프라인에 의존
- 종목 상세 뷰는 검색과 리스트 모두에서 진입점

## MVP Recommendation

**우선 구현:**
1. 상한가 근접 스캔 리스트 (임계값 조절 포함) — 핵심 가치 명제
2. 종목 현재가 + 등락률 + 거래량 — 기본 신뢰 구축
3. 종목별 뉴스 목록 (AI 요약 포함) — 차별화 포인트 1
4. 네이버 종목토론방 목록 (AI 요약 포함) — 차별화 포인트 2
5. 종목 검색 — 탐색 진입점

**연기:**
- 감성 수치 지표: 텍스트 레이블(긍정/부정/중립)로 단순화하여 v1에 포함 가능. 수치화는 v2
- 급등 + 거래량 복합 정렬: 등락률 단일 정렬로 v1 시작
- KOSPI 시장 지수 요약: 종목 리스트 상단에 간략 표시 고려, 단 선택적
