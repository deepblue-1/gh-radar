# Phase 08.1 Plan 06: webapp Switch 토글 Summary

`/stocks/[code]/discussions` 풀페이지에 shadcn Switch 기반 "의미있는 토론만 보기" 토글을 추가하고, `useSearchParams`/`router.replace` 로 URL 과 local state 를 sync 해 `?filter=meaningful`(기본 ON) / `?filter=all` 을 지원. 무한 스크롤의 `loadMore` 에도 filter 를 일관 전달하며, meaningful 결과 0건 시 안내 카피를 분기. 상세 Card(`stock-discussion-section.tsx`) 는 수정하지 않음.

## One-liner

useSearchParams + router.replace 로 URL sync 한 shadcn Switch 토글(기본 meaningful)을 풀페이지 전용으로 도입, fetchStockDiscussions 가 filter 를 load/loadMore 모두에 전달해 서버 08.1-02 계약(`filter=meaningful → relevance IS NULL OR != 'noise'`)을 소비 — production LoC +130 + test LoC +145, 143/143 webapp vitest PASS.

## Deviations from Plan

### 1. [Rule 3 — Blocker 회피] shadcn CLI 대신 직접 작성

- **Plan:** `pnpm --filter webapp dlx shadcn@latest add switch` 실행으로 `@radix-ui/react-switch` 의존성 자동 추가.
- **Actual:** `webapp/src/components/ui/switch.tsx` 를 기존 `toggle.tsx` / `popover.tsx` 와 동일한 스타일(`radix-ui` 통합 패키지 import) 로 직접 작성.
- **Reason:** webapp 은 이미 `"radix-ui": "^1.4.3"` 통합 패키지를 의존하고 있고, 해당 패키지의 `dist/index.d.mts` 가 `import * as reactSwitch from '@radix-ui/react-switch'; export { reactSwitch as Switch };` 로 Switch 를 re-export 한다. 기존 Toggle/Popover/Tooltip 7개 파일이 모두 같은 통합 패키지 패턴이라 shadcn CLI 가 추가했을 법한 개별 `@radix-ui/react-switch` 의존성을 더하는 것은 패턴 분열. `radix-ui` 통합 패키지만 유지해 의존성 최소화 + 기존 파일 일관성 우선.
- **Impact:** `grep -q "@radix-ui/react-switch" webapp/package.json` acceptance criterion 불만족(1/9). 그러나 기능적으로는 동일한 Radix 바이너리가 로드되며, shadcn CLI 가 실패하거나 pnpm 버전 호환 문제가 있을 때도 안정. 프로젝트 CLAUDE.md "기존 패턴 follow" 원칙 적용.
- **자가 평가:** Rule 4(architectural) 해당 안 함 — 설치 방식만 다를 뿐 같은 Radix Switch primitive. 사용자 대면 UX 차이 없음.

### 2. [Rule 3 — Blocker 회피] acceptance grep 경로와 실제 핸들러 경로 괴리
