## 수정 후 Error 클래스

```ts
export class NaverAuthError extends Error { name = 'NaverAuthError'; }           // 401 → cycle abort
export class NaverRateLimitError extends Error { name = 'NaverRateLimitError'; }  // 429 (신규) → per-stock backoff+retry, 최종 실패 시 종목만 skip
export class NaverBudgetExhaustedError extends Error { name = 'NaverBudgetExhaustedError'; }  // (예약) Naver 일일/월 quota exhausted 문구 감지 시만 사용 → cycle abort
export class NaverBadRequestError extends Error { name = 'NaverBadRequestError'; }  // 400/403 → per-stock skip
```

**429 분류 규칙 (본 phase 에서 확정)**: 기본적으로 `status === 429` → `NaverRateLimitError` throw. `NaverBudgetExhaustedError` 는 본 phase 에서 자동 throw 하지 않는다 (문자열 매칭 없이 보수적으로 reserve). 향후 Naver 가 명시적 budget-exhausted 문구를 응답에 넣거나 daily 카운터가 24,500 에 도달한 내부 상태에서만 throw 하도록 남겨둔다. → 본 phase 범위에서 `NaverBudgetExhaustedError` throw 경로는 코드에서 **제거**되지 않고 그대로 남되, 429 → `NaverRateLimitError` 로 대체된다.

## 429 backoff retry 스펙 (collectStockNews.ts 내부)

- page 호출이 `NaverRateLimitError` 를 던지면:
  1. `await sleep(250ms)` → re-call (page 재시도)
  2. 실패 시 `await sleep(500ms)` → re-call
  3. 두 retry 모두 실패 시 `NaverRateLimitError` propagate (per-stock 루프에서 종목 skip 처리)
- retry 간 page 파라미터(start/display/query) 동일 유지
- retry 는 `NaverRateLimitError` 에만 적용 — 다른 에러는 즉시 propagate
- **incrementUsage semantics**: `onPage` 콜백은 **page 호출이 최종 성공한 경우에만 1회 호출** 한다(현 구조 유지). retry 중간에는 호출하지 않음 — Naver 가 429 를 돌려주면 quota 카운터는 증가하지 않는 것이 정책. 따라서 `fetchPageWithRateLimitBackoff` 내부에서는 `onPage` 를 호출하지 않고, 기존 `collectStockNews` 페이지 루프가 성공 반환을 받은 뒤 1회 호출한다.

     * Phase 07.2 — page 호출을 429 backoff retry 로 감싼다.
     * NaverRateLimitError 만 retry (sleep 250ms → 500ms, 최대 2회).

    async function fetchPageWithRateLimitBackoff(
      client: import("axios").AxiosInstance,
      query: string,
      start: number,
      display: number,
    ): Promise<NaverNewsItem[]> {
      const delaysMs = [250, 500];  // 2 retries after first failure
      let lastErr: unknown;
      for (let attempt = 0; attempt <= delaysMs.length; attempt++) {
        try {
          return await searchNews(client, query, { start, display });
        } catch (err: unknown) {
          if (err instanceof NaverRateLimitError) {
            lastErr = err;
            if (attempt < delaysMs.length) {
              await new Promise((r) => setTimeout(r, delaysMs[attempt]));
              continue;
            }
            throw err;  // exhausted retries — propagate
          }
          throw err;  // non-rate-limit errors are not retried
        }
      }
      throw lastErr ?? new NaverRateLimitError();
    }
