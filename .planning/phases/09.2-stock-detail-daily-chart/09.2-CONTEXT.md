<domain>
## Phase Boundary

종목 상세페이지(`/stocks/[code]`)의 **Hero 바로 아래, StatsGrid 위** 영역에 해당 종목의 **일봉 캔들스틱 차트 + Volume bar** 를 전용 카드로 렌더한다. Phase 9 에서 적재된 `stock_daily_ohlcv` (4,003,432 행, 2020-01-02 ~ 현재) 를 single source 로 사용해 트레이더가 가격 흐름을 즉시 시각적으로 확인할 수 있게 한다.

**포함:**
- 일봉 캔들스틱 (양봉 빨강·음봉 파랑 한국식 색상 컨벤션 — Phase 3 `up/down` 토큰 재사용)
- 캔들 하단 Volume bar (대충 고정 — separate panel, 캔들과 동일 색상 정책)
- 기간 토글 (기본 1M, 옵션 1M / 3M / 6M / 1Y)
- 차트 카드: Hero ↓ / StatsGrid ↑ 사이 전용 섹션 (`stock-daily-chart-section` 명명 후보)
- 데이터: webapp → Supabase PostgREST 직접 호출 (Phase 06.2 watchlist 패턴 mirror)
- 로딩 스켈레톤 + 빈 종목 (신규 상장·폐지·데이터 결측) 처리
- RESEARCH 단계 POC — recharts (raw) / recharts (shadcn 래퍼) / lightweight-charts 3종 mockup 비교 후 최종 라이브러리 선정

**제외:**
- 분봉/주봉/월봉 — 일봉만
- 차트 위 이동평균선(MA) / RSI / MACD 등 보조지표 — v1 캔들 + Volume 만, MA 등은 후속
- 줌/팬/드래그 인터랙션 — v1 은 기간 토글 4종만 (확장 인터랙션은 후속)
- 차트 풀스크린/전용 페이지 (`/stocks/[code]/chart`) — v1 은 상세 페이지 내 섹션만
- 종목 비교 / 다중 종목 오버레이 — 별도 phase
- 수정주가 (adj_close) — Phase 9 D-04 raw close 정책 승계 (분할/증자 보정은 후속 phase)
- 인트라데이 실시간 캔들 — Phase 09.1 stock_daily_ohlcv 오늘자 row 가 STEP1/STEP2 로 매분 갱신되긴 하지만, 본 phase 의 캔들은 "일봉 완성" 시점 기준 표시 (EOD 17:30 candle-sync overlay 가 최종)

## Implementation Decisions

### 시각화 형태
- **D-01:** **캔들스틱(OHLC candle)** 채택. REQUIREMENTS Out of Scope 의 "캔들스틱 차트 — TradingView/키움이 이미 지배, 차별화 없음" 정책을 사용자가 명시적으로 뒤집음 (2026-05-15). 차별화 논리 대신 **상세 페이지 자체 완결성 + 트레이더가 화면 전환 없이 현재 종목의 가격 흐름을 즉시 파악** 을 우선. ⇒ REQUIREMENTS.md 갱신 follow-up 필수 (Out of Scope 행 삭제 + DATA-01 또는 신규 요구사항으로 일봉차트 표시 명시).
- **D-02:** 색상 컨벤션 = **한국식** (양봉 = `var(--up)` 빨강, 음봉 = `var(--down)` 파랑). Phase 3 디자인 토큰 재사용. 미국식(양봉 녹·음봉 적) 미채택 — 국내 트레이더 타겟.
- **D-03:** **Volume bar 동반 (대충 고정 위치)**. 캔들 패널 하단에 별도 panel (예: 캔들 패널 75% / Volume 25% 비율 권장, 정확값 planner 재량). bar 색상은 캔들 색상과 동기화 (양봉=빨강, 음봉=파랑) — 가격 방향과 거래량의 시각적 정합. MA / 보조지표 미포함 (v1).

### 기간 + 인터랙션
- **D-04:** **기본 기간 = 1M (~22 영업일)**. 캔들 폭 가독성 최적 구간 (영업일 너무 많으면 캔들 폭이 좁아져 OHLC 패턴 식별 불가). 사용자 명시 + RESEARCH 단계에서 캔들 폭 ≥4px 가 가독성 임계인지 검증.
- **D-05:** **기간 토글 4종 = 1M / 3M / 6M / 1Y**. 버튼 그룹 (segmented control 또는 shadcn `Tabs`/`ToggleGroup`). 3Y/All 제외 — 캔들 폭이 너무 좁아져 캔들 차트의 가치가 사라짐 (RESEARCH 단계 검증 후 필요 시 자동 주봉/월봉 전환 검토하되 본 phase 범위 아님).
- **D-06:** 줌/팬/드래그/툴팁 hover 정책: **툴팁 hover 만 v1 포함** (개별 캔들의 date/OHLC/Volume 표시). 줌/팬은 후속 phase. 모바일 터치 인터랙션은 lib 기본 동작에 위임 (RESEARCH POC 단계 확인).

### 차트 라이브러리
- **D-07:** **RESEARCH 단계에서 3개 라이브러리 mockup POC 비교 후 최종 결정** — 사용자 명시 (2026-05-15 1차 "recharts와 lightweight-charts 두개를 사용해서 그린 주식차트 목업을 확인해보고 싶어" + 2026-05-15 보강 라운드 "차트에 대한 목업디자인을 확인해보고 싶어. 기존 디자인 시스템을 따르면서, 실제 차트 라이브러리를 이용한 버전으로 (Next.js) 목업을 3개 만들어보자"). RESEARCH 가 실측해야 할 라이브러리 3종:
  1. **recharts (raw)** — 공식 Candlestick 컴포넌트 없음 → `Bar` + `Customized` shape 패턴으로 캔들 합성 + Volume bar 합성. 디자인 시스템 토큰(`var(--up)` / `var(--down)`) 수동 주입. 번들 크기 측정.
  2. **recharts + shadcn Chart 래퍼** ([ui.shadcn.com/charts](https://ui.shadcn.com/charts)) — 동일 recharts 기반이나 `<ChartContainer>` / `<ChartTooltip>` / `<ChartLegend>` 가 Phase 3 디자인 토큰을 자동 매핑. 캔들 합성은 raw recharts 패턴 그대로 + shadcn primitive 래핑 가능성 검증.
  3. **lightweight-charts 4.x** — `CandlestickSeries` + `HistogramSeries`(Volume) 네이티브 지원. 명령형 API → `useEffect` 경계 처리, 다크/라이트 토큰 주입 방식 검증.
  - 공통 검증 항목: 동일 fixture(예: 삼성전자 005930 1M 데이터)로 시각적 mockup 캡처, 다크/라이트 테마 호환성 (Phase 3 CSS 변수), SSR (Next.js App Router) 호환 — `'use client'` 경계 처리 (Phase 6 전체 클라이언트 페이지 정책 승계), 번들 영향 (현재 webapp 번들 baseline 대비 +KB).
- **D-08:** RESEARCH 산출물 = **세 라이브러리 mockup + 비교표**.
  - **Mockup 위치:** 격리 HTML 이 아니라 Next.js 라우트로 (`webapp/src/app/_mockups/chart-recharts/page.tsx`, `chart-shadcn/page.tsx`, `chart-lightweight/page.tsx` — 가칭, planner/researcher 재량). 이유: Phase 3 디자인 토큰(`var(--up)`, `var(--down)`, `Card`, `Skeleton`, 다크모드)이 자동 적용되어 "기존 디자인 시스템을 따르면서" 라는 사용자 요구를 충족하고, 실제 webapp 환경의 SSR / 번들 영향까지 함께 측정 가능.
  - **비교표 권장 컬럼:** 라이브러리 / 번들 +KB / 캔들+Volume 합성 난이도 / 디자인 토큰 통합도 / 다크모드 / 모바일 터치 / 툴팁 hover / SSR 호환 / 종합.
  - **선정 흐름:** 사용자가 3개 mockup 라우트를 직접 본 후 1개 선정 → planner 가 PLAN 에 lock-in. 본 CONTEXT 에서는 **라이브러리 선정 미확정**.

### 위치 / 레이아웃
- **D-09:** **차트 카드 위치 = Hero ↓ / StatsGrid ↑**. 사용자 명시 (2026-05-15). 별도 페이지·탭 UI 미채택 (scope creep 회피). 신규 컴포넌트 `webapp/src/components/stock/stock-daily-chart-section.tsx` (가칭) 를 `StockDetailClient` 의 Hero 바로 다음 자리에 마운트.
- **D-10:** 카드 컨테이너 = 기존 `Card` 컴포넌트 재사용 (Phase 3 디자인 시스템). 카드 헤더에 "일봉 차트" 타이틀 + 기간 토글 버튼 그룹. 카드 본문에 차트 영역 (고정 높이 권장, 예: `h-[360px]` — 정확값 planner 재량).
- **D-11:** 반응형 — 모바일(`<md`) 에서도 차트 가독성 유지. 모바일 캔들 최소 폭/높이 정책은 RESEARCH POC 에서 확인 후 planner 가 최종 명세.

### 데이터 fetch
- **D-12:** **webapp → Supabase PostgREST 직접 호출** (사용자 명시 2026-05-15). Phase 06.2 `webapp/src/lib/watchlist-api.ts` 패턴 mirror. 새 server Express 엔드포인트 추가 **하지 않음**. 사유: stock_daily_ohlcv 는 read-only · anon SELECT 허용 (Phase 9 D-03 RLS), 4M 행 중 1M 슬라이스는 PostgREST 가 RLS + 인덱스(date DESC) 로 자연 처리.
- **D-13:** 신규 모듈 `webapp/src/lib/daily-ohlcv-api.ts` (가칭) 추가:
  - `fetchDailyOhlcv(code: string, range: '1M' | '3M' | '6M' | '1Y', signal?: AbortSignal): Promise<DailyOhlcvRow[]>`
  - 내부 구현: `webapp/src/lib/supabase/client.ts` (브라우저 클라이언트) 사용 → `.from('stock_daily_ohlcv').select(...).eq('code', code).gte('date', from).lte('date', to).order('date', { ascending: true })`
  - `range` → `from` 계산: 1M = today − 60일 (영업일 변환 마진 고려, 22 영업일 ≈ 32 calendar 일이지만 보수적으로 60일 잡고 limit ~30 으로 클라이언트 슬라이스 권장), 3M / 6M / 1Y 동일 패턴
  - 반환 행 컬럼: `date`, `open`, `high`, `low`, `close`, `volume`, `trade_amount`, `change_amount`, `change_rate` (Phase 9 D-03 스키마)
- **D-14:** 캐싱 정책: **클라이언트 in-memory only (v1)** — 페이지 진입/refresh 마다 fetch. SWR/React Query 미도입 (현재 webapp 패턴 일관). 향후 동일 종목 재방문 시 캐시 hit 필요해지면 별도 phase 도입. `daily-ohlcv-api` 내부에 AbortController 만 적용 (이전 요청 취소 — Phase 6 패턴 승계).
- **D-15:** **공유 타입은 packages/shared 에 추가**. `packages/shared/src/dailyOhlcv.ts` (가칭): `DailyOhlcvRow` 인터페이스. Phase 9 의 `workers/candle-sync/src/types/stockDailyOhlcv.ts` 와의 타입 중복 / 동일화 정책은 planner 재량 (현재 Phase 9 타입은 worker 내부 전용 — webapp 노출 X).

### 로딩 / 에러 / Empty
- **D-16:** **Skeleton**: 차트 카드 영역에 `Skeleton` 컴포넌트 (Phase 3) — height 동일, animate-pulse. `StockDetailClient` 의 초기 로드 시 차트 섹션도 skeleton 표시. `StockDetailSkeleton` 확장 또는 차트 전용 skeleton.
- **D-17:** **Empty (데이터 결측 종목)**: stock_daily_ohlcv 에 해당 code 데이터가 0 행 (신규 상장 미수집 / 폐지 후 retention 만료 / KRX 응답 누락) → "일봉 데이터가 아직 수집되지 않았습니다" 안내 카드 (Phase 6 `ComingSoonCard` 재사용 또는 신규 `EmptyChartState`). 차트 카드 자체는 렌더하되 본문만 안내문구.
- **D-18:** **Error (Supabase fetch 실패)**: 인라인 에러 메시지 + "다시 시도" 버튼 (Phase 6 `StockDetailClient` 에러 패턴 mirror). 차트 카드 자체는 유지하여 stale-but-visible.

### Refresh / 갱신 정책
- **D-19:** **자동 폴링 없음** — Phase 6 D5 정책 승계. 일봉은 EOD 17:30 candle-sync 가 갱신 (Phase 9 D-09). 사용자가 페이지 새로고침 또는 `StockDetailClient` 의 "새로고침" 버튼 누를 때 같이 refetch. 차트 카드 내부 별도 refresh 버튼 미신설 (StockDetailClient refresh 가 차트 fetch 도 트리거).
- **D-20:** **인트라데이 갱신 의도적 미연결** — Phase 09.1 STEP1/STEP2 가 stock_daily_ohlcv 의 오늘자 row 를 매분 UPSERT 하지만, 차트는 어제까지 완성된 일봉만 표시(또는 오늘자 row 포함하되 사용자에게 "장중 미완 캔들" 표기는 본 phase 범위 아님). Planner 가 정책 명문화 (오늘자 row 포함/제외 + 표기). 권장: **오늘자 row 포함 + 라벨 "장중"** (트레이더가 가장 최근 흐름 즉시 확인 가능).

### 접근성 / 다크모드
- **D-21:** 차트는 `aria-label="일봉 차트"` + screen reader fallback (요약 텍스트: "최근 1개월 종가 흐름: 상승/하락 N영업일, 최고가/최저가/평균"). 시각 정보 외에 텍스트로도 핵심 정보 노출.
- **D-22:** 다크/라이트 테마 = Phase 3 CSS 변수 (`var(--up)`, `var(--down)`, `var(--flat)`, `var(--muted-fg)`, `var(--border)`) 차트 lib 에 주입. RESEARCH 단계에서 두 후보 lib 의 테마 토큰 주입 방식 비교.
