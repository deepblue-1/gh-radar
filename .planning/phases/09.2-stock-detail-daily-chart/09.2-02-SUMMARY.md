## 3. 다크모드 effect 패턴 코드 인용 (Pitfall 6 회귀 방지)

mockup3 단계에서 Post-Deploy 디버그로 발견된 결함: `document.documentElement.classList.contains('dark')` 가 useEffect mount 시점에만 evaluate 되어 사용자가 테마 토글해도 차트 색상이 안 바뀜.

**Fix — `stock-daily-chart.tsx`:**

```typescript
const { resolvedTheme } = useTheme(); // next-themes — 'light' | 'dark' | undefined

// 2) theme 변경 시 chart + series 색상 재주입 (Pitfall 6 fix)
useEffect(() => {
  const chart = chartRef.current;
  const candle = candleSeriesRef.current;
  const volume = volumeSeriesRef.current;
  if (!chart || !candle || !volume) return;
  const palette = getChartPalette(resolvePaletteKey(resolvedTheme));
  applyPaletteToChart(chart, candle, volume, palette);

  // Volume bar 의 per-bar color 도 현재 rows 기준 재주입 (theme 변경만으로
  //   rows effect 가 안 돌므로 직접 호출).
  if (rows.length > 0) {
    const volumeData: HistogramData<Time>[] = rows.map((r) => ({
      time: r.date as Time,
      value: r.volume,
      color: r.close >= r.open ? palette.up : palette.down,
    }));
    volume.setData(volumeData);
  }
}, [resolvedTheme, rows]);
```

**핵심 회귀 방지:**
- `useTheme()` 의 `resolvedTheme` 가 의존성에 들어가 React 가 변경을 감지 → effect 재실행
- `applyPaletteToChart` 가 chart/candle 양쪽에 새 palette 주입
- **Volume bar 는 series.applyOptions 만으론 부족** — color 가 per-bar (HistogramData.color) 로 지정되므로 `volume.setData` 도 재호출
- 색상 source 는 `getChartPalette` 단일 진입점 — 컴포넌트 내부 hex/oklch literal **0건**
