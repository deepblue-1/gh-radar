## Spacing Scale

기존 4-point 토큰 스케일 상속 (globals.css `--s-*`). 신규 값 없음.

| Token | Value | Usage (챗봇) |
|-------|-------|-------|
| --s-1 | 4px | 아이콘 갭, 인라인 미세 간격 |
| --s-2 | 8px | 메시지 내 요소 간격, composer 버튼 갭 |
| --s-3 | 12px | 시트 헤더/composer 패딩 |
| --s-4 | 16px | thread 패딩, 메시지 블록 간 세로 간격 |
| --s-5 | 24px | 상태 박스 패딩 |
| --s-6 | 32px | 섹션 구분 |
| --s-8 / --s-10 | 48 / 64px | 페이지 레벨 |

**Radius:** `--r`(6, 버튼) · `--r-md`(8, 입력/카드) · `--r-lg`(12, 시트/컨테이너) · 버블 12px · FAB/badge `9999px`.

Exceptions:
- **FAB**: 56px 높이 (터치 타깃), `9999px` radius — 표준 예외.
- **user 버블**: 비대칭 radius `12px 12px 4px 12px` (말꼬리 방향).
- **모바일 하단 시트**: 최대 높이 `85dvh`, thread 자동 스크롤.

## Typography

기존 타입 토큰 상속 (globals.css `--t-*`). 챗봇에서 실제 사용하는 롤만 명시.

| Role | Size | Weight | Line Height | 적용 |
|------|------|--------|-------------|------|
| Message body (answer) | 14px (`--t-sm`) | 400 | 1.6 | assistant 마크다운 본문 · user 버블 |
| Answer heading (h3) | 16px (`--t-base`) | 600 | 1.2 (`--lh-tight`) | 답변 내 소제목 |
| Label / meta | 12px (`--t-caption`) | 600 | 1.5 | 전문가 이름, 대화 타임스탬프, 출처, 표 헤더 |
| Sheet / conversation title | 14px (`--t-sm`) | 600 | 1.2 | 시트 헤더 제목, 대화 제목 |
| Micro (disclaimer, code) | 11px | 500 | 1.4 | 면책 문구, 종목코드 |
| 숫자 (가격/등락률) | `.mono` (Geist Mono, tabular-nums) | 600 | — | 미니 종목카드 가격, 표 숫자 셀 |

**Weights:** 표준 계약은 **regular 400 + semibold 600** 2단. (기존 디자인 시스템의 버튼 `font-medium 500`은 상속 유지 — 신규 UI에서 추가 도입 금지.)

한글 혼용 처리: `html[lang="ko"]` 전역 규칙 상속 — `word-break:keep-all` + `letter-spacing:-0.01em`. 숫자는 `.mono` (tabular-nums slashed-zero).

## Color

60/30/10 배분은 앱 전역 계약 상속. 챗봇 표면 매핑:

| Role | Token (light / dark) | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` (#FFFFFF / oklch .08) | thread 배경, 페이지 배경 |
| Secondary (30%) | `--card` · `--muted` · `--secondary` · `--popover` | 시트/카드 면, 대화목록, 진행표시 배경, 미니 종목카드, 출처 인용 블록 |
| Accent (10%) | `--primary` (oklch .63/.72 250) | 아래 목록 한정 |
| Semantic — 상승 | `--up` / `--up-bg` (빨강, oklch 22) | 미니 종목카드 상승 배지, 미니차트 상승봉, 표 양수 |
| Semantic — 하락 | `--down` / `--down-bg` (파랑, oklch 250) | 미니 종목카드 하락 배지, 미니차트 하락봉, 표 음수 |
| Destructive | `--destructive` (oklch .66 22) | 대화 삭제 확인만 |

**Accent(`--primary`) reserved for:** ① 전역 FAB 배경 ② composer 전송 버튼 ③ assistant 아바타(AI) ④ 진행표시 애니메이션 도트 / active 스텝 ⑤ `/chat` active 대화·nav 하이라이트(`--accent` 틴트) ⑥ textarea focus ring(`--ring`). 그 외 인터랙티브 요소(새 대화, 중단, outline 버튼)는 `--border`+`--muted` hover로 처리 — accent 남용 금지.

**국내 색상 관례 필수(D-07/specifics):** 수익·상승 = **빨강(`--up`)**, 손실·하락 = **파랑(`--down`)**. lightweight-charts 미니차트는 oklch 직접 주입 금지 — `chart-colors.ts`로 hex/rgb 변환 후 inject (memory lesson `feedback_lightweight_charts_oklch`).

| Login-required body | `AI 애널리스트는 로그인 후 이용할 수 있어요. 대화 기록은 계정에 안전하게 저장됩니다.` (버튼: `Google로 로그인`) |

| Error state | 제목 `답변을 불러오지 못했어요` · 본문 `일시적인 오류로 응답이 중단됐어요. 잠시 후 다시 시도해 주세요.` (버튼: `다시 시도`) |
| Partial-failure 고지 | 답변 본문 말미 `일부 전문가 의견을 가져오지 못해 가용한 정보로 답변했어요.` (Claude's Discretion — 에이전트 실패 시) |

| Destructive — 대화 삭제 | 트리거: 대화목록 🗑 아이콘 → 확인 다이얼로그 `이 대화를 삭제할까요?` / `삭제한 대화는 되돌릴 수 없어요.` / 버튼 `삭제`(destructive) · `취소`(outline) |

## Interaction Contract

- **로그인 게이트(D-01):** 비로그인 FAB/입력 클릭 → 로그인 필요 상태 박스. 체험 모드 없음.
- **종목 컨텍스트(D-03):** FAB이 `usePathname`으로 `/stocks/[code]` 감지 → 해당 종목 최근 대화 자동 이어가기, 없으면 새 대화. `＋ 새 대화`로 초기화.
- **스트리밍/중단(D-06):** 응답 중 정지 버튼 노출. 새 메시지 전송 시 이전 응답 자동 abort. 시트를 닫아도 서버는 응답 완료 후 저장.
- **진행 피드백(D-04/D-05):** 전문가 단계만 표기, 개별 의견 카드는 미노출. 팀장 종합 답변만.
- **접근성:** FAB `aria-label`, Sheet Radix 포커스 트랩/ESC 상속, active 대화 `aria-current`, 아이콘 `aria-hidden`, 진행표시 `aria-live="polite"`, 스트리밍 텍스트 영역 `aria-busy` 토글. 대비 WCAG AA(기존 토큰 검증됨).
- **반응형:** desktop 우측 드로어 / mobile 하단 시트. `/chat` 2-col → 1-col(`<640px`).

## Open Decision (목업 확인 필요)

**진행표시(C5) 변형** — 목업 상단 토글로 A/B 비교:
- **A · 인라인:** 단일 행 "실시간 검색 전문가 분석 중…" + 애니메이션 도트 (가벼움).
- **B · 스텝퍼:** 전문가별 done✓/active●/wait○ 세로 리스트 (진행감·대기 체감 완화 우수).

**연구자 권고 = B (스텝퍼).** D-04("에이전트별 단계 표시 … 5~20초 대기 체감 완화")의 의도를 가장 직접적으로 충족. D-05(개별 *의견* 미노출)와 충돌 없음 — 노출하는 건 "어느 전문가가 작업 중"이지 의견 내용이 아님. 본 UI-SPEC은 B로 박제했으며, 사용자가 목업에서 A 선호 시 C5/Copywriting만 A로 교체.
