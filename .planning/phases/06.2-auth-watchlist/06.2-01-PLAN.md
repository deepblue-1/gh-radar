## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ↔ webapp middleware | Next.js edge runtime 에서 요청 쿠키 파싱 · 응답 쿠키 설정 (HttpOnly 불가한 구간) |
| webapp middleware ↔ Supabase Auth | HTTPS 로 JWT 검증 + refresh rotation (Supabase 자체 PKCE) |
| Google OAuth ↔ Supabase ↔ webapp callback | 3-hop redirect chain — Google → Supabase project URL → webapp `/auth/callback?code=` |
| webapp `next` query param | 사용자 제어 입력이 리다이렉트 URL 에 사용됨 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.2-01 | Spoofing | `/auth/callback` OAuth code exchange | mitigate | Supabase `@supabase/ssr` PKCE state 쿠키 자동 검증 (code_verifier pair). webapp 은 `exchangeCodeForSession(code)` 호출만 담당 — 별도 state 검증 불필요. ASVS V2.1.1. Verified by: Plan 08 `e2e/specs/auth.spec.ts::login-redirect`. |
| T-06.2-02 | Tampering | Open redirect via `?next=` parameter | mitigate | `/auth/callback` route 에서 `rawNext.startsWith("/") && !rawNext.startsWith("//")` 가드 적용. protocol-relative URL (`//attacker.com`) 차단.
