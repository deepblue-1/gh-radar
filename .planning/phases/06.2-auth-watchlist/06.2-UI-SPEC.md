# Phase 06.2 — UI Design Contract

> Visual and interaction contract for Google OAuth 로그인 + 사이드바 유저 섹션 + `/watchlist` + ⭐ 토글.
> 기반: Phase 3 디자인 시스템 (webapp/src/styles/globals.css) · Phase 5 Scanner Table/Card · Phase 6 StockHero.
> 신규 토큰/색상 **추가 없음** — 기존 토큰 재사용만. 모든 신규 UI는 `--bg`/`--fg`/`--muted`/`--border`/`--primary`/`--destructive` + `--t-*`/`--s-*` 스케일 내에서 구현.

## Spacing Scale

Declared values (multiples of 4, 기존 `--s-*` 토큰 그대로):

| Token | Value | Usage (06.2 범위) |
|-------|-------|-------|
| `--s-1` | 4px | 아바타-텍스트 간격, ⭐ 아이콘 내부 패딩 |
| `--s-2` | 8px | 유저섹션 행 수직 패딩, 로그인 에러 카드 내부 padding |
| `--s-3` | 12px | 사이드바 항목 좌우 패딩 (기존 AppShell 패턴 계승) |
| `--s-4` | 16px | 기본 갭 — 유저섹션 내부 column gap, 사이드바 nav 항목 상하 gap |
| `--s-5` | 24px | Login Card 내부 섹션 간격, Watchlist empty state 수직 gap, AppShell `main` 기본 padding |
| `--s-6` | 32px | Login 페이지 Card 와 화면 경계 사이 수직 여백 |
| `--s-8` | 48px | Login Card 의 Google 버튼 아래 여유 공간 (CTA 고립) |
| `--s-10` | 64px | Watchlist empty state 컨테이너 최소 높이 여백 (min-h-[240px] 연장 시) |

- ⭐ Toggle 버튼 터치 타겟: **최소 36px × 36px** (WCAG 2.5.5 AA, Table 행 `--row-h=36px` 과 수직 align). `shadcn Toggle size="sm"` 는 32px — Phase 06.2 는 `size="default"` (36px) 를 Table 행 내 사용, Scanner 행 hover 시 시각적 추가 gap 없음.
- 사이드바 유저섹션 상단 구분선: 1px (`border-t` `--border-subtle` — 기존 카드 inner highlight 토큰 재사용).

## Typography

기존 `--t-*` 토큰만 사용 — 신규 크기/웨이트 도입 없음. 사이즈 4종 + 웨이트 2종으로 제한.

| Role | Token | Size | Weight | Line Height | Usage (06.2 범위) |
|------|-------|------|--------|-------------|-------|
| Caption | `--t-caption` | 12px | 600 semibold | `--lh-tight` 1.2 | 에러 라벨, 유저섹션 "로그아웃" micro-label (옵션) |
| Body SM | `--t-sm` | 14px | 400 regular | `--lh-normal` 1.5 | Login description ("Google 계정으로 로그인"), 유저섹션 이메일, Watchlist empty 보조 문구, Scanner 행 코드/캡션 |
| Body | `--t-base` | 16px | 400 regular | `--lh-normal` 1.5 | Login CTA 버튼 label, Watchlist empty heading, 종목명 |
| Heading H3 | `--t-h3` | 20px | 600 semibold | `--lh-tight` 1.2 | Login Card title ("gh-radar 로그인"), `/watchlist` 페이지 h1 |

Weights: `400` regular (본문) · `600` semibold (강조/제목) — **두 가지만**.
숫자/코드는 `.mono` utility (`Geist Mono` + `tabular-nums` + `ss01 slashed-zero`) 필수 — Watchlist 표시 컬럼의 현재가/등락률/거래대금/코드 전부.

## Color

60/30/10 split은 기존 Toss 팔레트를 그대로 승계한다. 06.2는 새 색상 도입 없음.

| Role | Token | Light | Dark | Usage |
|------|-------|-------|------|-------|
| Dominant (60%) | `--bg` | `#FFFFFF` | `oklch(0.08 0 0)` | 페이지 배경, `main` 영역 배경 |
| Secondary (30%) | `--muted` / `--card` | `oklch(0.96 0 0)` / `#FFFFFF` | `oklch(0.18 0 0)` / `oklch(0.12 0 0)` | 사이드바(`--muted`), Login Card(`--card`), 유저섹션 컨테이너, Table thead, Watchlist empty 컨테이너 |
| Accent (10%) | `--primary` | `oklch(0.63 0.18 250)` blue | `oklch(0.72 0.16 250)` | (아래 "reserved for" 참조) |
| Destructive | `--destructive` | `oklch(0.66 0.20 22)` red | `oklch(0.72 0.19 22)` | 로그인 에러 메시지 배경(`--destructive/10` tint)·텍스트, 50개 초과 경고 |
| Up | `--up` | = destructive red | = destructive red | Watchlist 행 등락률 양수 (Scanner와 동일) |
| Down | `--down` | = primary blue | = primary blue | Watchlist 행 등락률 음수 (Scanner와 동일) |
| Flat | `--flat` | `oklch(0.55 0 0)` | `oklch(0.65 0 0)` | Watchlist 행 변동 0 |

**Accent (`--primary`) reserved-for list — 06.2 범위:**
1. Login 페이지의 **Google 로그인 버튼 focus-visible ring** (기본 Button default variant 의 primary bg 사용 — Card CTA 로 고립되어 있음)
2. 사이드바 네비 **선택된 항목 active 상태** (pathname === item.href — `bg-[var(--accent)]` + `text-[var(--accent-fg)]` · accent 계열 서브 토큰 재사용, primary 직접 사용 금지)
3. ⭐ Toggle **pressed 상태 아이콘 fill** (pressed 일 때 `text-[var(--primary)]` + `fill-[var(--primary)]` — 관심종목 등록 시각화)
4. Watchlist empty state 의 **"Scanner로 가기" 링크 버튼** (primary variant)

⛔ 금지: Watchlist Table 일반 행·배경·테두리·텍스트 본문·유저섹션 아바타 기본색에 `--primary` 사용 금지. Up/Down 은 독립 토큰 (`--up`/`--down`) 사용, primary 와 시각적으로 같더라도 의미 분리.

**Destructive reserved-for list — 06.2 범위:**
1. Login 페이지 `?error=*` 에러 알림 카드 (`bg-[var(--destructive)]/10` + `text-[var(--destructive)]`)
2. Watchlist 50개 초과 시 inline 경고 + 토스트(있다면 — Discretion: 07번 참조)
3. 사이드바 유저섹션의 **로그아웃 버튼 hover 시 텍스트** — `text-[var(--muted-fg)] hover:text-[var(--destructive)]` (위험 연상이 아닌 정중한 ghost → destructive 전이)

### 5. ⭐ Toggle 시각 규칙 (2026-04-16 목업 리뷰 확정 — Ghost variant)

**shadcn `Toggle` 컴포넌트 + Ghost variant (D-26):**

- **Unset (관심종목 아님):** `<Star className="size-4" strokeWidth={1.5} />` + 부모 `text-[var(--muted-fg)]/70 hover:text-[var(--primary)]` + 배경 없음

## Accessibility Contract

- 전역 focus-ring: 기존 `globals.css §8.5.5` double-ring 패턴 유지 (2px outline + 4px softer shadow).
- ⭐ Toggle: `aria-pressed={isWatched}` (Radix Toggle 자동) + `aria-label` 한글 (종목명 포함).
- 로그인 에러 알림: `role="alert"` + 한글 설명 텍스트. `aria-live="polite"`.
- 사이드바 nav: `<nav aria-label="주 메뉴">` + active 링크 `aria-current="page"`.
- UserSection: 로그아웃 버튼 `aria-label="로그아웃"` (icon-only 인 경우).
- Watchlist empty: `role="status"` + heading + CTA 순서 — screen reader 가 제목 → 설명 → 액션 순 읽기 가능.
- 모든 `lucide-react` 아이콘: 의미 없는 경우 `aria-hidden="true"`, 의미 있는 경우 `aria-label` 명시.
- WCAG 2.5.5 AA 터치 타겟: ⭐ Toggle 36×36, 로그아웃 아이콘 버튼 `size="icon"`(36×36), Login Google 버튼 `size="lg"`(44px).
- 키보드 네비: Tab 순서 — 헤더 로고 → ⌘K 서치 → 테마토글 → 사이드바 nav 항목들 → UserSection 아바타 skip → 로그아웃 버튼 → main content.
