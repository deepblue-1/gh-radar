### Plan 본문과의 차이점

**WatchlistToggle `localPressed` fallback 3-tier** — 플랜 action 예시는 `localPressed ?? (initialPressed ?? pressedFromSet)`. 구현은 `localPressed ?? initialPressed ?? pressedFromSet` (동일) + 서버 재동기화 후 `useEffect` 로 localPressed 를 null 로 리셋 — 플랜에 명시 없었지만 멀티 탭 동기화에 필요. 에러 시에는 localPressed 를 명시적으로 false/true 로 재설정하여 로컬이 권위.

**Scanner Table 셀별 Link 접근성 처리** — 플랜은 "실행자 재량" 명시. 선택 방식:
- 첫 셀 (종목명): 의미있는 Link, `aria-label` 명시
- 나머지 5 셀: 동일 href 이지만 `tabIndex={-1}` + `aria-hidden="true"` 로 스크린리더/키보드 중복 차단 — 시각적으로는 행 전체 클릭 가능하되 접근성 관점에선 "하나의 행=하나의 링크". 

**기존 `stock-detail-client.tsx` 의 사전-실패 테스트는 변경 없음** — 플랜 범위 밖. Plan 05 SUMMARY 가 기록한 대로 `toBeInTheDocument` jest-dom matcher 파일별 loading 이슈로, 본 Plan 이 해결할 문제가 아님.
