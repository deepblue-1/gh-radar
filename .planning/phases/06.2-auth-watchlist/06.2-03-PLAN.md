## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser ↔ middleware | 비로그인/로그인 상태 기반 분기 결정 |
| /login 페이지 iframe embedding | 공격자 사이트가 /login 을 iframe 에 넣어 clickjacking 시도 |
| `?next=` 쿼리 파라미터 | 사용자 제어 문자열이 redirect URL 경로에 삽입 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.2-11 | Tampering | Open redirect via `?next=` parameter | mitigate | `/login` 페이지의 `safeNext` 가드 (`startsWith("/")` + `!startsWith("//")`) + `/auth/callback` 의 동일 가드 (Plan 01). 이중 방어. ASVS V4.1.1, V5.1.5. Verified by: Plan 08 `e2e/specs/auth.spec.ts::open-redirect-guard` (planner 추가 예정). |
| T-06.2-12 | Tampering | Clickjacking on /login | mitigate | `X-Frame-Options: DENY` + `Content-Security-Policy: frame-ancestors 'none'` on `/login` and `/auth/*`. ASVS V14.4.7. Verified by: Task 3 acceptance_criteria (response header smoke test). |
| T-06.2-13 | Elevation of Privilege | Middleware bypass via path traversal or encoded paths | mitigate | Next.js URL 정규화 자동 적용 (`request.nextUrl.pathname`). `config.matcher` 는 `_next/static` 제외하지만 사용자 입력 경로는 모두 통과. ASVS V4.1.3. |
| T-06.2-14 | Spoofing | 피싱 사이트가 gh-radar 로 위장한 login 페이지 유도 | accept | 사용자 교육 영역. CSP + HTTPS only (Vercel 기본) 로 위조 도메인 식별 지원. 실질 방어는 Google OAuth 공식 도메인 consent screen. |
| T-06.2-15 | Information Disclosure | `?error=*` 값 변조로 내부 상태 추측 | accept | 4종 에러 외 fallback `unknown` 메시지 — 공격자가 얻을 정보 없음. 로그에 에러 키 기록 시 민감정보 아님. |
