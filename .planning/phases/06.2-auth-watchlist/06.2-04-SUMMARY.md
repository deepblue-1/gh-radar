# Phase 06.2 Plan 04: AppShell 사이드바 활성화 + AppSidebar + UserSection Popover Summary

Radix Popover 기반 UserSection 과 AppSidebar 를 신규 추가하여 `/scanner`·`/stocks/[code]` 에 사이드바 shell 을 활성화하고, D-16 팝오버 UX 와 D-17 (/design 회귀 방지) 계약을 충족한다.

## What was built

### 신규 컴포넌트 2종

**`webapp/src/components/layout/user-section.tsx` — UserSection**
- **트리거** (`PopoverTrigger asChild`): `<button>` 아바타(size-8) + 이름(max-w-[140px] truncate) + `aria-haspopup="menu"`.
- **팝오버 콘텐츠** (`PopoverContent side="top" align="start"`): `w-56 p-2 bg-[var(--card)] border-[var(--border)] shadow-sm`.
  - 상단 헤더: 아바타(size-10) + 이름(`--t-sm semibold`) + 이메일(`--t-caption muted-fg truncate`).
  - `<Separator className="my-1" />`.
  - 로그아웃 Button `variant="ghost" size="sm"` + `<LogOut />` + "로그아웃" + `hover:text-[var(--destructive)]` + `aria-label="로그아웃"`.
- **아바타 fallback 체인**:
  1. `user.user_metadata.avatar_url` (Google 프로필) → `<img onError>` 발동 시 `setImgError(true)` 상태 잠금.
  2. 잠금 또는 URL 부재 시 이메일 로컬파트 첫 글자 대문자 이니셜 (`bg-[var(--muted-fg)]/15` 배경).
- **signOut**: `useAuth()` 의 helper 호출 (Plan 01 AuthProvider 가 `supabase.auth.signOut()` 후 `/login` hard redirect 수행).
- **방어**: `if (!user) return null` — bfcache/초기 로딩 1~2 프레임 보호.

**`webapp/src/components/layout/app-sidebar.tsx` — AppSidebar**
- `<nav aria-label="주 메뉴">` + `flex h-full flex-col justify-between`.
- `NAV` 상수 배열로 2 항목 정의: `/scanner` (Activity), `/watchlist` (Star).
- `usePathname()` 기반 active 감지 → `aria-current="page"` + `bg-[var(--accent)] text-[var(--accent-fg)] font-semibold`.
- `data-nav-item` 속성으로 app-shell.tsx 의 Sheet drawer 자동 닫힘 hook 에 연동 (기존 로직 재사용).
- 하단 `<UserSection />` 배치.
