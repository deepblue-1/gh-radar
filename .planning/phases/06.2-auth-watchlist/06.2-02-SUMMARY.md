## What was built

**마이그레이션 파일 `supabase/migrations/20260416120000_watchlists.sql` (105 lines):**

- `watchlists` 테이블 — PK `(user_id, stock_code)` + FK `auth.users(id)` ON DELETE CASCADE + FK `stocks(code)` ON DELETE CASCADE + `added_at` DEFAULT now() + `position` int nullable
- 인덱스 `idx_watchlists_user_added_at` (user_id, added_at DESC) — 기본 정렬 키 커버
- RLS 4정책 (authenticated role 전용):
  - `auth_select_own_watchlists` — `auth.uid() = user_id` USING
  - `auth_insert_own_watchlists` — `auth.uid() = user_id` WITH CHECK
  - `auth_update_own_watchlists` — USING + WITH CHECK
  - `auth_delete_own_watchlists` — USING
- PL/pgSQL 함수 `enforce_watchlist_limit()` — 51번째 row INSERT 시 `RAISE EXCEPTION 'watchlist_limit_exceeded' ERRCODE = 'P0001'`
- BEFORE INSERT trigger `trg_enforce_watchlist_limit` (FOR EACH ROW)
- `stocks` 테이블: `anon_read_stocks_master` → `read_stocks_master` rename + `TO anon, authenticated` 확장
- `stock_quotes` 테이블: `anon_read_stock_quotes` → `read_stock_quotes` rename + `TO anon, authenticated` 확장
