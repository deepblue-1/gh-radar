## Decisions Made

- **쿠키 포맷 `base64-<base64url>`**: supabase/ssr 0.10.x 의 공식 writer 포맷과 일치. reader 는 prefix 유무로 auto-detect 하지만 writer 포맷을 따라 안정성 확보. Buffer.toString("base64") → `+`→`-`/`/`→`_`/`=` 제거로 base64url 변환.
- **service_role E2E 격리**: `SUPABASE_SERVICE_ROLE_KEY` 는 env 기반 seed/cleanup helper 에만 전파. webapp runtime 은 anon key 만 사용 (Threat T-06.2-36 미티게이션).
- **50-limit seed 종목 범위**: 005930 (삼성전자) 부터 50연번 (005979). stocks/stock_quotes JOIN 없이 watchlists row count 만 50 보장. UI 에서 quote 없음 → "—" 렌더이지만 limit 동작은 유효.
- **미인증 describe 이중 방어**: `test.use({ storageState })` + `beforeEach context.clearCookies()` — fullyParallel worker reuse 시 발생할 수 있는 쿠키 leak 대비.

**2. [Rule 1 - Bug] 쿠키 포맷을 supabase/ssr 공식 `base64-<base64url>` 로 변경**

- **Found during:** Task 3 E2E 최초 실행 (authed describe 전체 실패)
- **Issue:** 초기 구현은 `encodeURIComponent(JSON.stringify(session))` — supabase/ssr 0.10.x reader 가 지원하지만 writer 는 `base64-` prefix 포맷을 사용. 비표준 포맷이 middleware 의 getUser() 흐름에서 간헐적 검증 실패 유발 가능.
- **Fix:** `Buffer.from(JSON.stringify(session),"utf-8").toString("base64")` → base64url 치환 (`+→-`, `/→_`, `=` 제거) → `base64-` prefix 붙여 쿠키 값으로 저장.
- **Files modified:** `webapp/e2e/auth.setup.ts`
- **Verification:** setup project 재실행 green + `.playwright/auth.json` 의 쿠키 value 가 `base64-...` 로 시작하는지 확인.
- **Committed in:** `defe3ca` (Task 3)

**3. [Rule 2 - Missing Critical] 미인증 describe 에 `beforeEach clearCookies()` 추가**

- **Found during:** Task 3 E2E 재실행 (미인증 케이스가 쿠키를 가지고 있는 이상 동작 관찰)
- **Issue:** fullyParallel worker reuse 환경에서 `test.use({ storageState: { cookies: [], origins: [] } })` 만으로는 격리가 불완전할 수 있음.
- **Fix:** `test.beforeEach(async ({ context }) => { await context.clearCookies() })` 이중 방어 추가.

- **Verification:** tsc clean. (런타임 검증은 Deferred 참조)
