## Decisions Made

- **rebuildTopMovers marketMap 인자 추가 (Rule 1 — Bug)**: PLAN 원안은 `rows = top.map((u, idx) => ({ rank, code, updated_at }))` 만 INSERT 하지만, `top_movers` 스키마는 `name text NOT NULL`, `market text NOT NULL CHECK (KOSPI|KOSDAQ)` 제약. PLAN 그대로 실행 시 production INSERT 실패. 본 plan 이 marketMap 인자를 추가하고 step1Updates.name (옵셔널, fallback code) + marketMap.get(code) (fallback KOSPI) 로 payload 보강. test 도 marketMap fixture 추가.
- **top_movers DELETE 패턴 `.neq("code", "")` 채택**: PLAN 원안 `.gte("rank", 0)` 은 rank=NULL row 가 있으면 매칭 안 되어 일부 stale row 잔류 가능. `code` 는 PRIMARY KEY NOT NULL 이라 `.neq("code", "")` 가 항상 매칭 → 전체 row 안전 삭제. (PostgreSQL `text != ''` 는 NOT NULL 보장)
- **휴장일 가드 (RESEARCH §2.5)**: 0 row → warn + `return { step1Count: 0, step2Count: 0, failed: 0 }` (process.exit 0 정상). < MIN_EXPECTED=1500 → `throw new Error("ka10027 N < 1500 — partial response")` (Cloud Run Job retry + alert 발생). 0 row 와 partial 분리해서 alert 노이즈 회피.
- **dedupe Map by code (RESEARCH §3.3.4)**: ka10027 페이지네이션 응답에 페이지 경계 중복 row 가 등장할 수 있음 (실측 미관찰, 가설). `Map<string, IntradayCloseUpdate>` 에 `set(code, u)` 호출하면 마지막 row 가 자동 승리 → for-loop 후 `Array.from(dedupeMap.values())` 로 unique 추출. test 는 미커버 (가능성 가설), 운영 시 logger.info `{ mapped, mapErrors }` 로 추적 가능.
- **market map 산출 (stocks 마스터 join)**: STEP1 mapping 후 `step1Updates.map(u => u.code)` 를 `from("stocks").select("code, market").in("code", codes)` 로 조회. KOSPI/KOSDAQ 만 marketMap 에 push (CHECK 제약 충족). 신규 상장으로 master-sync 가 못 따라잡은 종목은 fallback KOSPI (bootstrapMissingStocks 가 KOSPI 로 등록한 것과 일치).
- **STEP1/STEP2 컬럼 분리 UPSERT (T-09.1-21 mitigate)**: stock_quotes 의 `onConflict: "code"` 가 페이로드 컬럼만 UPDATE 한다는 Supabase JS 동작 의존. STEP1 (price/change/volume/trade_amount + code/name/market/updated_at) 과 STEP2 (open/high/low/upper_limit/lower_limit/market_cap + code/updated_at) 가 서로 다른 컬럼 집합 → 자연 race-free. test 에서 payload 컬럼 검증 (price 가 STEP2 페이로드에 없음 / open 이 STEP1 페이로드에 없음).
- **CHUNK_SIZE=1000 일관**: candle-sync 의 upsertOhlcv 와 동일 (RESEARCH §7 T-09-07). PostgREST jsonb 페이로드 제한 회피. 1,898 row → 2 chunk (1000 + 898) 자연 분할. 250 row → 단일 chunk.
