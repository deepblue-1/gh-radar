## Decisions Made

- **candle-sync MODE dispatch 의도적 제거**: intraday-sync 는 단일 cycle 워커이므로 candle-sync 의 `Mode = "backfill" | "daily" | "recover"` 분기가 불필요. `config.ts` 가 candle-sync 보다 backfillFrom/backfillTo/recoverLookback/recoverThreshold/recoverMaxCalls/basDd 6 필드 생략 + parseMode 함수 부재.
- **redact path 7개 채택 (RESEARCH §9.3)**: T-09.1-07 (Information Disclosure) 차단 — KIWOOM 측 5개 (kiwoomAppkey, kiwoomSecretkey, headers.authorization, token, access_token, accessToken) + Supabase 공통 1개 (supabaseServiceRoleKey). wildcard `*.` prefix 로 top-level + nested object 동시 흡수 (candle-sync 패턴 승계).
- **ka10001 rate limit 24 req/s default**: CONTEXT D-12 (사용자 2026-05-13 실측) 채택. 보수 20 req/s 대신 실측값으로 hot set ~250 calls / cycle ≈ 10.5초 시간 예산 보장. env override 가능 (`KA10001_RATE_LIMIT`).
- **MIN_EXPECTED_ROWS=1500 (candle-sync 1400 ↑)**: CONTEXT D-06 의 활성 ~1,898 종목 가정 + 휴장일/거래정지 자연 감소 흡수 여유. T-09.1-08 (config) mitigate 의 일환으로 휴장일 빈 응답 guard 와 정상 cycle 구분.
- **HOT_SET_TOP_N=200 default**: CONTEXT D-11 (등락률 상위 200 ∪ watchlist unique) — Plan 06 의 runIntradayCycle 이 사용. watchlist unique 수 (현재 수십~수백) 와 합쳐 ~250 calls / cycle 예상.
- **.dockerignore baseline = ingestion 패턴**: candle-sync 에 .dockerignore 부재라 workers/ingestion 의 패턴을 baseline 으로 채택 + `tests/*.test.ts` 명시 제외 + `!.env.example` 예외 추가. Wave 1 vitest 파일이 production image 에 포함되지 않도록 보장.
