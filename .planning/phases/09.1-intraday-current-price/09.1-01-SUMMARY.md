## Decisions Made

- **RPC #1 의 open 의도적 omit**: 09:00 첫 cycle INSERT 시에는 `close` 임시값으로 초기화하지만, 이후 cycle 의 ON CONFLICT branch 에서는 `open` 컬럼을 UPDATE 하지 않음. RPC #2 (hot set) 가 ka10001 정확 시가로 덮어쓰고, hot set 외 종목은 EOD 17:30 candle-sync 가 KRX 공식값으로 최종 overlay (D-33, T-09.1-03).
- **RPC #2 의 close/volume/trade_amount/change_amount/change_rate 의도적 omit**: STEP1 이 매분 갱신하는 컬럼이라 STEP2 가 ka10001 응답값(0 또는 NULL)으로 덮어쓰면 데이터 손실. ON CONFLICT 의 SET 절을 `open/high/low/inserted_at` 4개로만 제한 (D-34, T-2).
- **kis_tokens DROP 시점 분리**: 본 plan 에 파일은 생성하지만 Wave 0 push 대상에서 제외. Wave 4 의 KIS 폐기 cutover 마지막 step (Cloud Scheduler PAUSE → Job 삭제 → SA/Secret 정리 후) 에서만 production 적용 (T-09.1-04).
- **REVOKE 3줄 명시 패턴**: `REVOKE FROM PUBLIC` 단독으로는 Supabase 플랫폼 auto-grant 가 `anon, authenticated` 에 재부여하는 회귀 발생. `PUBLIC + anon,authenticated` 둘 다 REVOKE + `service_role` 만 GRANT 의 3줄 패턴 필수 (MEMORY feedback_supabase_rpc_revoke, T-09.1-01, T-09.1-02).
