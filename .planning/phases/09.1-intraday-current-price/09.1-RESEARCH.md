## Summary

본 phase 는 KIS ingestion 의 **완전 폐기** 와 키움 REST API 2종 endpoint (ka10027 + ka10001) 기반 단일 ingestion 파이프라인 정착이 핵심이다. 이전 RESEARCH (2026-05-13, "STEP1 close-only / KIS 공존") 와 비교해 5개 영역이 확장/재정의됐다:

1. **2-step cycle** — STEP1 ka10027 페이지네이션 (~700ms, 활성 1,898) → STEP2 ka10001 hot set 순차 (~10.5s, ~250 종목). 매분 cycle 총 ~13초 (60s task-timeout 내 매우 여유).
2. **RPC 2종** — `intraday_upsert_close(jsonb)` (STEP1 close/change/volume + 09:00 첫 INSERT 시 OHL 임시) + `intraday_upsert_ohlc(jsonb)` (STEP2 hot set 의 정확 OHLC + 상한/하한/시가총액 덮어쓰기). 두 RPC 모두 service_role only REVOKE 명시.
3. **stock_quotes client-side UPSERT** — RPC 미사용. STEP1 (1,898 × 6 컬럼) + STEP2 (~250 × 6 컬럼) 순차 호출, dedupe Map 으로 충돌 회피.
4. **Cloud Run service 도 VPC connector** — server (`gh-radar-server`) 가 키움 ka10001 동기 호출하므로 동일 Static IP 경유. 재배포 필요 (현재 default network).
5. **KIS 완전 폐기** — workers/ingestion 디렉터리 통째 삭제, server/src/kis → server/src/kiwoom 교체, packages/shared/src/kis.ts 삭제, kis_tokens 테이블 drop migration, Cloud Scheduler/Job/SA/Secret 정리.

**Primary recommendation:** PLAN 은 candle-sync 워크스페이스 1:1 mirror + server/src/kis 의 patterns(client/tokenStore/inquirePrice) 를 server/src/kiwoom 으로 재작성 + supabase migration 2개 (kis_tokens drop + kiwoom_tokens create + RPC 2종) 통합. KIS 폐기는 **마지막 Wave** 에 배치 — production rollover 가 검증된 후 (Cloud Scheduler PAUSE → smoke green → 삭제 순서).

**Remaining risks (Plan 단계 fixture 캡처 / [BLOCKING] task):**
- **R1:** `now_trde_qty` 누적성 — Plan 02 fixture 의 2회 연속 호출 비교로 확정 (가설 99% — KIS `acml_vol` 컨벤션)
- **R2:** `mac` 단위 가설 (억원) — Plan 02 fixture 캡처에서 시가총액 ÷ (price × listing_shares) 검증
- **R3:** 키움 IP 화이트리스트 등록 — Static IP reserve 후 사용자 액션 ([BLOCKING] Wave 마지막)
- **R4:** Cloud Run service 의 VPC 마이그레이션 재배포 시 cold-start latency — min-instances=1 유지로 자연 보완

### 1.4 부호 Prefix 의미

**[VERIFIED: 사용자 실측 2026-05-13]**

- `cur_prc`, `pred_pre`, `flu_rt` 모두 첫 글자가 `+` (상승) / `-` (하락) / `0` (보합) 또는 부호 생략.

- `pred_pre`, `flu_rt` 는 **부호 유지** — 음수 음수 그대로 DB 저장. `change_rate numeric(10,4)` 가 음수 허용.

### 1.7 Rate Limit

**판정: HIGH — 사용자 실측 2026-05-13**

- ka10027 (등락률 순위): 매 cycle 10 페이지 + 토큰 1회 = 11 calls/min ≈ 0.2 calls/sec → 안전.
- ka10001 (단일 종목): ~24 req/s, 429 코드 `[1700:허용된 요청 개수를 초과]`. STEP2 hot set ~250 calls = 10.5초.
- 키움 rate limit 은 **IP 단위 통합 bucket 가설** — 사용자/계좌 단위 아니라 IP 단위. Static IP 1개 공유 (worker + server) → 통합 bucket.
- 결과적 budget: 1초당 24 calls 중 worker STEP2 가 평균 24 calls 사용, server on-demand 는 1분 1회 미만 → 충분 여유.

**Pitfall:** server 의 종목 상세 페이지 traffic burst 시 (사용자 다수 접속) 동시 호출이 worker STEP2 와 겹치면 429. **mitigation: ka10001 client 에 token-bucket rate limiter 24 req/s 적용 (worker + server 각각)**.

### 1.8 IP 화이트리스트 요구사항

**판정: HIGH — 사용자 명시 + CONTEXT D-04, D-29, D-31**

- 키움 REST API 는 등록된 IP 에서만 호출 가능.
- 본 phase 는 Cloud Run Job (intraday-sync) **+ Cloud Run service (gh-radar-server)** 둘 다 동일 Static IP 경유 (D-04, D-29).
- 사용자 액션: Static IP reserve 후 키움 openapi.kiwoom.com 의 IP 등록 페이지에 1개 등록 (D-31, [BLOCKING] Wave).

### 2.2 Mapper 함수 (STEP 1)

```typescript
// workers/intraday-sync/src/pipeline/map.ts
import type { KiwoomKa10027Row, IntradayCloseUpdate } from "@gh-radar/shared";

export function ka10027RowToCloseUpdate(
  row: KiwoomKa10027Row,
  dateIso: string,
): IntradayCloseUpdate {
  const code = row.stk_cd.replace(/_AL$/, "");
  if (!/^\d{6}$/.test(code)) {
    throw new Error(`Invalid stk_cd after strip: "${row.stk_cd}" → "${code}"`);
  }

  const price = parseSignedPrice(row.cur_prc).value;
  const changeAmount = parseOptionalSignedNumber(row.pred_pre);
  const changeRate = parseOptionalSignedNumber(row.flu_rt);
  const volume = parseOptionalNumber(row.now_trde_qty) ?? 0;

  // D-23: trade_amount = volume × close 근사값 (트레이딩 시그널 용도, 회계 아님)
  const tradeAmount = Math.round(volume * price);

  return {
    code,
    date: dateIso,
    name: row.stk_nm,
    price,
    changeAmount,
    changeRate,
    volume,
    tradeAmount,
  };
}

export function parseSignedPrice(s: string | undefined): { value: number; direction: "up" | "down" | "flat" } {
  if (!s) throw new Error("missing signed price");
  const trimmed = s.trim();
  const sign = trimmed[0];
  const rest = (sign === "+" || sign === "-") ? trimmed.slice(1) : trimmed;
  const abs = Number(rest.replace(/,/g, ""));
  if (!Number.isFinite(abs)) throw new Error(`invalid signed price: "${s}"`);
  return {
    value: abs,
    direction: sign === "+" ? "up" : sign === "-" ? "down" : "flat",
  };
}

export function parseOptionalSignedNumber(s: string | undefined): number | null {
  if (!s || s.trim() === "") return null;
  const n = Number(s.replace(/,/g, ""));
  return Number.isFinite(n) ? n : null;
}

export function parseOptionalNumber(s: string | undefined): number | null {
  if (!s || s.trim() === "") return null;
  const n = Number(s.replace(/,/g, ""));
  return Number.isFinite(n) ? n : null;
}
```

## 3. UPSERT 정책 — 2개 RPC + client-side stock_quotes

### 3.1 RPC #1 — `intraday_upsert_close(jsonb)` (STEP 1)

STEP1 의 1,898 종목 close/change/volume + 09:00 첫 cycle INSERT 시 OHL 임시값 처리.

```sql
-- supabase/migrations/{ts}_intraday_upsert_close_rpc.sql
BEGIN;

CREATE OR REPLACE FUNCTION public.intraday_upsert_close(
  p_rows jsonb
)
RETURNS integer
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  r jsonb;
  affected integer := 0;
BEGIN
  FOR r IN SELECT * FROM jsonb_array_elements(p_rows)
  LOOP
    INSERT INTO public.stock_daily_ohlcv
      (code, date, open, high, low, close, volume, trade_amount, change_amount, change_rate, inserted_at)
    VALUES (
      r->>'code',
      (r->>'date')::date,
      (r->>'close')::numeric(20,2),    -- INSERT 시: open = cur_prc (D-33)
      (r->>'close')::numeric(20,2),    -- INSERT 시: high = cur_prc
      (r->>'close')::numeric(20,2),    -- INSERT 시: low = cur_prc
      (r->>'close')::numeric(20,2),
      (r->>'volume')::bigint,
      (r->>'trade_amount')::bigint,    -- D-22, D-23: volume × close 근사값
      NULLIF(r->>'change_amount', '')::numeric(20,2),
      NULLIF(r->>'change_rate', '')::numeric(10,4),
      now()
    )
    ON CONFLICT (code, date) DO UPDATE
      SET close         = EXCLUDED.close,
          high          = GREATEST(stock_daily_ohlcv.high, EXCLUDED.close),
          low           = LEAST(stock_daily_ohlcv.low, EXCLUDED.close),
          volume        = EXCLUDED.volume,
          trade_amount  = EXCLUDED.trade_amount,
          change_amount = EXCLUDED.change_amount,
          change_rate   = EXCLUDED.change_rate,
          inserted_at   = now();
          -- open: 의도적 omit (첫 INSERT 만 set)
          -- STEP2 의 intraday_upsert_ohlc 가 hot set 의 정확 open/high/low 로 덮어쓰기
    affected := affected + 1;
  END LOOP;
  RETURN affected;
END;
$$;

REVOKE EXECUTE ON FUNCTION public.intraday_upsert_close(jsonb) FROM PUBLIC;
REVOKE EXECUTE ON FUNCTION public.intraday_upsert_close(jsonb) FROM anon, authenticated;
GRANT EXECUTE ON FUNCTION public.intraday_upsert_close(jsonb) TO service_role;

COMMIT;
```

### 3.2 RPC #2 — `intraday_upsert_ohlc(jsonb)` (STEP 2, 신규)

STEP2 의 hot set ~250 종목 정확 OHLC + 상한가/하한가/시가총액. STEP1 의 임시값 (open=high=low=close) 위에 정확값 덮어쓰기.

**중요:** RPC #2 는 항상 RPC #1 **후** 호출 (STEP1 의 임시 row INSERT 가 선행되어야 ON CONFLICT 의 UPDATE branch 진입).

```sql
-- supabase/migrations/{ts}_intraday_upsert_ohlc_rpc.sql
BEGIN;

CREATE OR REPLACE FUNCTION public.intraday_upsert_ohlc(
  p_rows jsonb
)
RETURNS integer
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  r jsonb;
  affected integer := 0;
BEGIN
  FOR r IN SELECT * FROM jsonb_array_elements(p_rows)
  LOOP
    INSERT INTO public.stock_daily_ohlcv
      (code, date, open, high, low, close, volume, trade_amount, change_amount, change_rate, inserted_at)
    VALUES (
      r->>'code',
      (r->>'date')::date,
      (r->>'open')::numeric(20,2),
      (r->>'high')::numeric(20,2),
      (r->>'low')::numeric(20,2),
      (r->>'open')::numeric(20,2),     -- INSERT 만 발생할 가능성 거의 없음 (STEP1 이 선행), 안전상 open 으로
      0,                                -- volume — STEP1 이 갱신
      0,                                -- trade_amount — STEP1 이 갱신
      NULL,
      NULL,
      now()
    )
    ON CONFLICT (code, date) DO UPDATE
      SET open  = EXCLUDED.open,        -- ka10001 의 정확 시가로 덮어쓰기 (D-14, D-34)
          high  = EXCLUDED.high,        -- ka10001 의 정확 고가
          low   = EXCLUDED.low,         -- ka10001 의 정확 저가
          inserted_at = now();
          -- close, volume, trade_amount, change_amount, change_rate: 의도적 omit
          -- STEP1 이 매분 갱신하는 컬럼들 — STEP2 가 덮어쓰면 안 됨
    affected := affected + 1;
  END LOOP;
  RETURN affected;
END;
$$;

REVOKE EXECUTE ON FUNCTION public.intraday_upsert_ohlc(jsonb) FROM PUBLIC;
REVOKE EXECUTE ON FUNCTION public.intraday_upsert_ohlc(jsonb) FROM anon, authenticated;
GRANT EXECUTE ON FUNCTION public.intraday_upsert_ohlc(jsonb) TO service_role;

COMMIT;
```

**호출 측:**

```typescript
  code: string;
  name: string;                 // ka10027 stk_nm (대부분 동일하므로 마스터에서 join 불필요)

  change_rate: number | null;
  volume: number;
  trade_amount: number;         // D-23 근사값

};

const CHUNK_SIZE = 1000;

export async function upsertStockQuotesStep1(
  supabase: SupabaseClient,
  rows: StockQuoteUpdate[],
): Promise<{ count: number }> {
  let total = 0;
  for (let i = 0; i < rows.length; i += CHUNK_SIZE) {

      .from("stock_quotes")

    total += chunk.length;
  }
  return { count: total };
}
```

| `price` | ✓ | — |
| `change_amount` | ✓ | — |
| `change_rate` | ✓ | — |
| `volume` | ✓ | — |
| `trade_amount` | ✓ (D-23) | — |
| `open` | — | ✓ (D-14) |
| `high` | — | ✓ |

### 3.4 stocks 마스터 FK orphan 회피

`stock_daily_ohlcv` 는 `stocks(code)` 에 FK NOT VALID (Phase 9 D-03). ka10027 응답의 활성 종목 ~1,898 + ka10001 hot set ~250 은 모두 stocks 마스터에 존재한다고 가정 — 하지만 신규 상장 종목이 master-sync `10 8` 전에 키움에 먼저 등장하면 FK 위반 가능.

### 3.5 RPC 호출 순서 보장 (cycle 내부)

매 cycle 의 순서:
1. STEP 1 — 키움 ka10027 페이지네이션 (~700ms)
2. STEP 1 — bootstrapStocks (missing codes)
3. STEP 1 — `intraday_upsert_close` RPC (chunked) → `stock_daily_ohlcv` 1,898 row UPSERT
4. STEP 1 — `stock_quotes` client-side UPSERT (chunked) → 1,898 row × 8 컬럼
5. STEP 1 — `top_movers` 재구성 (등락률 상위 60 추출 + stale cleanup — 본 RESEARCH §11 또는 별도 mapper 에서)
6. STEP 2 — hot set 산출 (top 200 ∪ watchlist) — §11 참조
7. STEP 2 — ka10001 순차 호출 (~10.5초, 24 req/s rate limiter)
8. STEP 2 — `intraday_upsert_ohlc` RPC → `stock_daily_ohlcv` ~250 row open/high/low UPDATE
9. STEP 2 — `stock_quotes` client-side UPSERT (~250 row × 6 다른 컬럼)

**총 시간:** STEP1 ~2-3초 (DB 포함) + STEP2 ~12-15초 = **15-18초**. 60s task-timeout 내 안전 (D-12).

**[CITED: docs.cloud.google.com/run/docs/configuring/vpc-direct-vpc]**

### 5.3 토큰 발급 + cache 패턴 (worker / server 공통)

```typescript
// 공통 — workers/intraday-sync/src/kiwoom/tokenStore.ts (server 도 동일 패턴 mirror)
import { SupabaseClient } from "@supabase/supabase-js";
import axios from "axios";

const TOKEN_REFRESH_THRESHOLD_MS = 5 * 60 * 1000; // 만료 5분 전 갱신 (D-27)

export type KiwoomToken = {
  accessToken: string;
  expiresAt: Date;
};

export async function getKiwoomToken(
  supabase: SupabaseClient,
  config: { kiwoomBaseUrl: string; kiwoomAppkey: string; kiwoomSecretkey: string; tokenType?: string },
): Promise<KiwoomToken> {
  const tokenType = config.tokenType ?? "live";

  // 1. 캐시 확인
  const { data: cached } = await supabase
    .from("kiwoom_tokens")
    .select("access_token, expires_at")
    .eq("token_type", tokenType)
    .maybeSingle();

  if (cached) {
    const expiresAt = new Date(cached.expires_at);
    const remainMs = expiresAt.getTime() - Date.now();
    if (remainMs > TOKEN_REFRESH_THRESHOLD_MS) {
      return { accessToken: cached.access_token, expiresAt };
    }
  }

  // 2. 만료 임박 또는 캐시 부재 → 새 토큰 발급
  const res = await axios.post(
    `${config.kiwoomBaseUrl}/oauth2/token`,
    { grant_type: "client_credentials", appkey: config.kiwoomAppkey, secretkey: config.kiwoomSecretkey },
    { timeout: 10_000, headers: { "content-type": "application/json" } },
  );

  const expiresAt = parseKiwoomExpiresDt(res.data.expires_dt);
  const accessToken = res.data.token;

  // 3. UPSERT (race condition idempotent — 마지막 INSERT 승)
  await supabase
    .from("kiwoom_tokens")
    .upsert(
      { token_type: tokenType, access_token: accessToken, expires_at: expiresAt.toISOString(), fetched_at: new Date().toISOString() },
      { onConflict: "token_type" },
    );

  return { accessToken, expiresAt };
}

function parseKiwoomExpiresDt(s: string): Date {
  // "20260513093013" → Date (KST → UTC)
  if (!/^\d{14}$/.test(s)) throw new Error(`Invalid expires_dt: ${s}`);
  const y = Number(s.slice(0, 4));
  const mo = Number(s.slice(4, 6)) - 1;
  const d = Number(s.slice(6, 8));
  const h = Number(s.slice(8, 10));
  const mi = Number(s.slice(10, 12));
  const se = Number(s.slice(12, 14));
  return new Date(Date.UTC(y, mo, d, h - 9, mi, se));
}
```

### 5.4 동시 refresh race condition (D-27)

- worker (1분 cycle) + server (사용자 요청 trigger) 가 동시 만료를 감지하고 둘 다 새 토큰 발급 시:
- 키움 OAuth endpoint 가 동일 토큰 반환 또는 새 토큰 발급 (둘 다 가능 — KIS 동작은 동일 토큰 반환, 키움 미검증)
- DB INSERT: 마지막 INSERT 가 승 (upsert `onConflict: token_type`). 양쪽 모두 정상 토큰 보유 → 다음 호출에서 fresh row 사용.

**결론:** Idempotent — 별도 lock 불필요.

### 5.5 ingestion 의 tokenStore.ts 패턴 mirror

기존 `workers/ingestion/src/kis/tokenStore.ts` (Phase 1) 가 동일 패턴 (Supabase 테이블 + 만료 시 refresh). kis_tokens 의 schema 와 1:1 mirror 로 kiwoom_tokens 작성하면 logic 거의 동일 (table 이름 + endpoint URL 만 교체). Plan 단계에서 ingestion tokenStore 를 참고용으로 활용 (디렉터리 폐기 전 git diff 비교).

### 8.2 Threat Model

| Pattern | STRIDE | Standard Mitigation |
|---|---|---|
| **T-1: KIWOOM_APPKEY/SECRETKEY 노출** | Information Disclosure | Secret Manager 저장 + Cloud Run `--set-secrets` 사용 + Cloud Logging redact (pino redact paths 확장 — `*.kiwoomAppkey`, `*.kiwoomSecretkey`, `headers.authorization`, `*.access_token`) + .env.deploy gitignore |
| **T-2: STEP1 / STEP2 동시 동일 row UPSERT race** | Tampering | RPC #1 / #2 가 **다른 컬럼 집합** 처리. 동일 row 의 close/volume (STEP1) ↔ open/high/low (STEP2) 가 서로 덮어쓰지 않음. stock_quotes 도 컬럼 분리 (§3.3.3) |
| **T-3: stock_daily_ohlcv open/high/low 무단 overwrite by STEP1** | Tampering | RPC #1 의 ON CONFLICT UPDATE 에 open 의도적 omit. high=GREATEST, low=LEAST 단조 갱신 → STEP2 가 정확값으로 덮어쓰기 → EOD 17:30 가 최종 overlay |
| **T-4: 키움 token race (worker / server 동시 만료)** | DoS | Supabase upsert idempotent — 마지막 INSERT 승. 양쪽 모두 정상 토큰 보유 |
| **T-5: Static IP 누설 → 키움 IP whitelist 우회 시도** | Spoofing | Static IP 는 reserved external IP, GCP 내부 routing. Cloud NAT egress 만. ingress 차단 (Job 은 ingress 없음, service 는 Cloud Run native auth) |
| **T-6: RPC `intraday_upsert_close/ohlc` 의 anon/authenticated 호출** | Elevation of Privilege | REVOKE FROM anon, authenticated 명시. GRANT service_role only (사용자 메모리 룰: feedback_supabase_rpc_revoke) |
| **T-7: candle-sync, master-sync 과의 충돌** | Concurrency | 시간대 분리 (D-38) + 컬럼 분리 (close/change/volume = intraday + candle-sync, open/high/low = STEP2 hot set + candle-sync EOD overlay) |
| **T-8: Cloud Run Job task-timeout 60s 초과** | DoS / data loss | task-timeout 60s + STEP1+STEP2 = ~15-18초 → 4x 여유. timeout 1 cycle 손실은 다음 cycle 회복 (idempotent UPSERT) |
| **T-9: 키움 API 응답 schema 변경** | Tampering (upstream) | MIN_EXPECTED_ROWS 가드 + Cloud Monitoring alert (5분 연속 실패 → email) |
| **T-10: 09:00 첫 cycle 의 Direct VPC Egress 1분+ 시작 지연** | Availability | 두 번째 cycle 부터 안정. RPC #1 INSERT 옵션 D 의 open=high=low=close 자연 보완 |
| **T-11: KIS 폐기 transition risk — 폐기 중 데이터 공백** | Availability | **순서 보장 절차 (§12):** 1) intraday-sync 가 production 에서 5분간 ka10027 cycle 성공 검증 → 2) Cloud Scheduler `gh-radar-ingestion-scheduler` PAUSE → 3) 10분 후 데이터 정합 확인 (stocks 마스터 무손상, stock_quotes 갱신 정상) → 4) Job/SA/Secret 정리 → 5) git commit 으로 코드/migration 삭제 |
| **T-12: server VPC 마이그레이션 시 cold-start latency spike** | Availability | min-instances=1 유지. 재배포 후 모니터링 (Cloud Logging latency p99). 실패 시 즉시 rollback (gcloud run services update --network=default) |
| **T-13: server kiwoom client 의 ka10001 동시 호출 burst (사용자 다수 접속)** | DoS (key=IP) | server 에 token-bucket rate limiter 24 req/s 적용. burst 초과 시 429 응답 + Retry-After |
| **T-14: 키움 응답의 mac 단위 가설 오류 (R2)** | Tampering (silent data corruption) | Plan 02 fixture 캡처로 단위 확정 후 mapper 곱셈 상수 고정. production 첫 cycle 후 시가총액 sanity check (3-5 종목 sample) |

## 11. Hot Set 산출 알고리즘 (D-11)

### 11.1 Hot Set 정의

**Hot set** = (STEP1 ka10027 응답에서 등락률 상위 200) ∪ (`watchlists.stock_code` unique 전체)

- ka10027 응답이 `sort_tp="1"` (상승률 내림차순) 정렬되어 있으므로 → **추가 정렬 불필요**. `.slice(0, 200)` 으로 top 200 추출.
- 모든 사용자 watchlist 의 unique 종목 — service-role client 로 RLS 우회 SELECT.
- 합집합 (`Set<string>`) 으로 중복 제거. 결과 평균 ~250 종목 (사용자가 적은 초기에는 ~200 종목).

### 11.2 구현

```typescript
// workers/intraday-sync/src/pipeline/hotSet.ts
import type { SupabaseClient } from "@supabase/supabase-js";
import type { IntradayCloseUpdate } from "@gh-radar/shared";

export async function computeHotSet(
  supabase: SupabaseClient,
  step1Updates: IntradayCloseUpdate[],
  topN: number,
): Promise<string[]> {
  // 1. STEP1 응답에서 top N (이미 등락률 내림차순 정렬)
  const topMovers = step1Updates.slice(0, topN).map(u => u.code);

  // 2. watchlists 전체 unique
  const { data: watchlistRows, error } = await supabase
    .from("watchlists")
    .select("stock_code");

  if (error) throw error;

  const watchlistCodes = (watchlistRows ?? []).map(r => r.stock_code as string);

  // 3. 합집합 (Set 중복 제거)
  const union = new Set<string>([...topMovers, ...watchlistCodes]);

  return Array.from(union);
}
```

### 11.3 Performance

- `watchlists.select('stock_code')` — Phase 06.2 의 watchlists 테이블, 현재 row 수 < 1,000 추정. 단일 SELECT ~수십 ms.
- Set 합집합 ~상수 시간 (< 1ms).
- 결과 250 종목 × 1/24s rate limit = 10.5초.

### 11.4 Edge cases

- **사용자 0명 (watchlists 빈 테이블):** hot set = top 200 만. 정상 동작.
- **사용자 watchlist 중 keys 가 stocks 마스터에 없음 (이론상 불가, FK가 있다면):** ka10001 호출이 정상 응답 안 할 가능성 → fail-isolation 으로 한 종목 skip. cycle 무손상.
- **top 200 ∪ watchlist 가 250 초과 (인기 종목 다수 watchlist):** 합집합 자연 처리, 250+ 종목도 10.5초 × N/250 비례 증가. 60s 안에 있으면 OK. 비상 시 `HOT_SET_TOP_N` env 로 top 200 → 150 으로 축소 가능.

### 13.2 client.ts (token-bucket rate limiter 24 req/s)

```typescript
// server/src/kiwoom/client.ts
import axios, { type AxiosInstance } from "axios";

const tokens = new Map<string, { available: number; lastRefill: number }>();
const BUCKET_KEY = "kiwoom-ka10001";
const BUCKET_CAPACITY = 24;
const REFILL_RATE_PER_SEC = 24;

export function acquireKiwoomRateToken(): Promise<void> {
  return new Promise((resolve) => {
    const tick = () => {
      const now = Date.now();
      const bucket = tokens.get(BUCKET_KEY) ?? { available: BUCKET_CAPACITY, lastRefill: now };
      const elapsedSec = (now - bucket.lastRefill) / 1000;
      bucket.available = Math.min(BUCKET_CAPACITY, bucket.available + elapsedSec * REFILL_RATE_PER_SEC);
      bucket.lastRefill = now;

      if (bucket.available >= 1) {
        bucket.available -= 1;
        tokens.set(BUCKET_KEY, bucket);
        resolve();
      } else {
        tokens.set(BUCKET_KEY, bucket);
        setTimeout(tick, 50);
      }
    };
    tick();
  });
}

export function createKiwoomClient(baseUrl: string): AxiosInstance {
  return axios.create({
    baseURL: baseUrl,
    timeout: 10_000,
    headers: { "content-type": "application/json;charset=utf-8" },
  });
}
```

### 13.3 inquirePrice.ts (server)

```typescript
// server/src/kiwoom/inquirePrice.ts
import type { AxiosInstance } from "axios";
import type { KiwoomKa10001Row } from "@gh-radar/shared";
import { acquireKiwoomRateToken } from "./client";

export async function fetchInquirePrice(
  client: AxiosInstance,
  token: string,
  code: string,
): Promise<KiwoomKa10001Row> {
  await acquireKiwoomRateToken();

  return res.data as KiwoomKa10001Row;
}
```

### Pattern 1: ka10027 페이지네이션 (STEP 1)

```typescript
// workers/intraday-sync/src/kiwoom/fetchRanking.ts
import type { AxiosInstance } from "axios";
import type { KiwoomKa10027Row } from "@gh-radar/shared";

const HARD_CAP = 5000;

export async function fetchKa10027(
  client: AxiosInstance,
  token: string,
): Promise<KiwoomKa10027Row[]> {
  const body = {
    mrkt_tp: "000",
    sort_tp: "1",
    trde_qty_cnd: "0000",
    stk_cnd: "0",
    crd_cnd: "0",
    updown_incls: "1",
    pric_cnd: "0",
    trde_prica_cnd: "0",
  };

  const all: KiwoomKa10027Row[] = [];
  let contYn: "Y" | "N" = "N";
  let nextKey: string | undefined;

  do {
    const headers: Record<string, string> = {
      authorization: `Bearer ${token}`,
      "api-id": "ka10027",
      "content-type": "application/json;charset=utf-8",
      "cont-yn": contYn,
    };
    if (nextKey) headers["next-key"] = nextKey;

    let res;
    try {
      res = await client.post<{ pred_pre_flu_rt_upper?: KiwoomKa10027Row[] }>(
        "/api/dostk/rkinfo", body, { headers },
      );
    } catch (err: any) {
      if (err?.response?.status === 401) throw new Error("키움 401 — token/credential 실패");
      if (err?.response?.status === 429) throw new Error("키움 429 — rate limit");
      throw err;
    }

    all.push(...(res.data.pred_pre_flu_rt_upper ?? []));
    contYn = (res.headers["cont-yn"] === "Y") ? "Y" : "N";
    nextKey = res.headers["next-key"];

    if (all.length >= HARD_CAP) throw new Error(`pagination hard cap ${HARD_CAP} exceeded`);
  } while (contYn === "Y" && nextKey);

  return all;
}
```

### Pattern 2: ka10001 hot set 순차 호출 (STEP 2)

```typescript
// workers/intraday-sync/src/kiwoom/fetchHotSet.ts
import type { AxiosInstance } from "axios";
import type { KiwoomKa10001Row } from "@gh-radar/shared";
import { acquireRateToken } from "./rateLimiter";

export async function fetchKa10001ForHotSet(
  client: AxiosInstance,
  token: string,
  codes: string[],
): Promise<{ successful: KiwoomKa10001Row[]; failed: number }> {
  const results = await Promise.allSettled(
    codes.map(async (code) => {
      await acquireRateToken();  // 24 req/s token bucket
      const headers = {
        authorization: `Bearer ${token}`,
        "api-id": "ka10001",
        "content-type": "application/json;charset=utf-8",
      };
      const res = await client.post("/api/dostk/stkinfo", { stk_cd: code }, { headers });
      if (res.data.return_code !== 0) throw new Error(`ka10001 ${code}: ${res.data.return_msg}`);
      return res.data as KiwoomKa10001Row;
    })
  );

  const successful = results
    .filter((r): r is PromiseFulfilledResult<KiwoomKa10001Row> => r.status === "fulfilled")
    .map(r => r.value);
  const failed = results.filter(r => r.status === "rejected").length;

  return { successful, failed };
}
```

### Pattern 3: stock_quotes UPSERT (STEP 1 + STEP 2, client-side)

```typescript
// workers/intraday-sync/src/pipeline/upsertQuotes.ts
import type { SupabaseClient } from "@supabase/supabase-js";

const CHUNK = 1000;

export async function upsertQuotesStep1(
  supabase: SupabaseClient,
  rows: Array<{ code: string; name: string; market: string; price: number; change_amount: number | null; change_rate: number | null; volume: number; trade_amount: number; updated_at: string }>,
): Promise<{ count: number }> {
  let total = 0;
  for (let i = 0; i < rows.length; i += CHUNK) {
    const chunk = rows.slice(i, i + CHUNK);
    const { error } = await supabase.from("stock_quotes").upsert(chunk, { onConflict: "code" });
    if (error) throw error;
    total += chunk.length;
  }
  return { count: total };
}

export async function upsertQuotesStep2(
  supabase: SupabaseClient,
  rows: Array<{ code: string; open: number; high: number; low: number; upper_limit: number | null; lower_limit: number | null; market_cap: number | null; updated_at: string }>,
): Promise<{ count: number }> {
  // 250 행 < 1000 chunk → 단일 호출
  const { error } = await supabase.from("stock_quotes").upsert(rows, { onConflict: "code" });
  if (error) throw error;
  return { count: rows.length };
}
```

### Pattern 4: cycle index.ts (STEP1 + STEP2 통합)

```typescript
// workers/intraday-sync/src/index.ts
import "dotenv/config";
import { loadConfig } from "./config";
import { logger } from "./logger";
import { getKiwoomToken } from "./kiwoom/tokenStore";
import { createKiwoomClient } from "./kiwoom/client";
import { fetchKa10027 } from "./kiwoom/fetchRanking";
import { fetchKa10001ForHotSet } from "./kiwoom/fetchHotSet";
import { ka10027RowToCloseUpdate } from "./pipeline/map";
import { ka10001RowToOhlcUpdate } from "./pipeline/mapOhlc";
import { computeHotSet } from "./pipeline/hotSet";
import { rebuildTopMovers } from "./pipeline/topMovers";
import { bootstrapMissingStocks } from "./pipeline/bootstrapStocks";
import { intradayUpsertClose } from "./pipeline/upsertClose";
import { intradayUpsertOhlc } from "./pipeline/upsertOhlc";
import { upsertQuotesStep1, upsertQuotesStep2 } from "./pipeline/upsertQuotes";
import { createSupabaseClient } from "./services/supabase";
import { withRetry } from "./retry";

function todayIsoKst(): string {
  const now = new Date();
  const kst = new Date(now.getTime() + 9 * 60 * 60 * 1000);
  return `${kst.getUTCFullYear()}-${String(kst.getUTCMonth() + 1).padStart(2, "0")}-${String(kst.getUTCDate()).padStart(2, "0")}`;
}

export async function runIntradayCycle(): Promise<{ step1Count: number; step2Count: number; failed: number }> {
  const config = loadConfig();
  const supabase = createSupabaseClient(config);
  const dateIso = todayIsoKst();
  const log = logger.child({ dateIso });

  log.info("intraday cycle start");

  // 0. Token
  const token = await withRetry(() => getKiwoomToken(supabase, config), "getKiwoomToken");

  // STEP 1 — ka10027 페이지네이션
  const kiwoom = createKiwoomClient(config);
  const ka10027Rows = await withRetry(() => fetchKa10027(kiwoom, token.accessToken), "fetchKa10027");
  log.info({ rows: ka10027Rows.length }, "STEP1 ka10027 fetched");

  if (ka10027Rows.length === 0) {
    log.warn("ka10027 0 rows — 휴장일");
    return { step1Count: 0, step2Count: 0, failed: 0 };
  }
  if (ka10027Rows.length < config.minExpectedRows) {
    throw new Error(`ka10027 ${ka10027Rows.length} < ${config.minExpectedRows} — partial`);
  }

  // STEP 1 — mapping + bootstrap + upsert
  await withRetry(() => bootstrapMissingStocks(supabase, ka10027Rows), "bootstrapMissingStocks");
  const step1Updates = ka10027Rows
    .map(r => { try { return ka10027RowToCloseUpdate(r, dateIso); } catch { return null; } })
    .filter((u): u is NonNullable<typeof u> => u !== null);

  // STEP 1 — stock_daily_ohlcv RPC + stock_quotes + top_movers
  const { count: step1Count } = await withRetry(() => intradayUpsertClose(supabase, step1Updates), "intradayUpsertClose");
  await withRetry(() => upsertQuotesStep1(supabase, step1Updates.map(toQuoteRowStep1)), "upsertQuotesStep1");
  await withRetry(() => rebuildTopMovers(supabase, step1Updates), "rebuildTopMovers");

  // STEP 2 — hot set 산출
  const hotSet = await computeHotSet(supabase, step1Updates, config.hotSetTopN);
  log.info({ hotSetSize: hotSet.length }, "STEP2 hot set computed");

  // STEP 2 — ka10001 순차 호출
  const { successful: ka10001Rows, failed } = await fetchKa10001ForHotSet(kiwoom, token.accessToken, hotSet);
  log.info({ successful: ka10001Rows.length, failed }, "STEP2 ka10001 fetched");

  const step2Updates = ka10001Rows
    .map(r => { try { return ka10001RowToOhlcUpdate(r, dateIso); } catch { return null; } })
    .filter((u): u is NonNullable<typeof u> => u !== null);

  // STEP 2 — stock_daily_ohlcv RPC + stock_quotes
  await withRetry(() => intradayUpsertOhlc(supabase, step2Updates), "intradayUpsertOhlc");
  await withRetry(() => upsertQuotesStep2(supabase, step2Updates.map(toQuoteRowStep2)), "upsertQuotesStep2");

  log.info({ step1Count, step2Count: step2Updates.length, failed }, "intraday cycle complete");
  return { step1Count, step2Count: step2Updates.length, failed };
}

async function main(): Promise<void> {
  try {
    const out = await runIntradayCycle();
    logger.info({ ...out }, "intraday-sync complete");
    process.exit(0);
  } catch (err) {
    logger.error({ err }, "intraday-sync failed");
    process.exit(1);
  }
}

if (process.argv[1] && process.argv[1].endsWith("index.js")) main();
```
