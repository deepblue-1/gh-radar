## Decisions Made

- **인덱서 (`[key:string]: string`) 의도적 미사용**: kis.ts 의 `KisRankingRow` 는 schema 변화 흡수 목적으로 인덱서 포함. kiwoom raw 타입은 ka10001 의 45 필드 중 본 phase 가 명시적으로 사용하는 11 필드 (Ka10027 11 + Ka10001 11) 만 채택. 새 필드 필요 시 본 타입 확장 (R3 — Plan 04 fixture 캡처 후 결정). 타입 안전성 + 명시성 우선.
- **signed string 보존**: 키움의 `cur_prc / pred_pre / flu_rt` 는 첫 글자가 부호 (`+`, `-`, 빈값). raw 타입은 string 그대로 보존하고, mapper (Plan 04) 가 `parseSignedPrice(s): { price, direction }` 로 분해 (D-09). flu_rt 와 pred_pre 는 부호 유지하여 DB 저장.
- **mac 단위 가설 = 억원 (R2)**: KIS `hts_avls` 컨벤션과 동일하리라 가정. Plan 04 의 fixture 캡처 task 가 키움 실제 응답에서 단위를 확정. 가설 틀려도 mapper 의 `parseMac` 곱셈 상수 1줄 변경으로 해결 가능 → 본 타입 변경 불필요. `IntradayOhlcUpdate.marketCap` 의 주석에 "mac × 10^8" 명시.
- **test 가 `../kiwoom.js` 직접 import**: workspace re-export 경로 (`@gh-radar/shared`) 우회. vitest 4.x 의 ESM 모드는 ts 소스에서 직접 컴파일 검증 가능 — dist 빌드 산출물 의존 없음. PLAN spec 예시 그대로 따름.
