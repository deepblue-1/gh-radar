**스코프 IN:**
- 키움 REST API `ka10027` (등락률순위) 페이지네이션으로 활성 종목 batch 수집
- 매분 Cloud Run Job 실행, 응답 → stock_daily_ohlcv.close (+ change_amount/change_rate/volume) UPSERT
- open/high/low 는 candle-sync EOD 가 채움. intraday 동안에는 close 만 매분 갱신 (Phase 9 의 EOD row 가 base)

- 새 워커 `workers/intraday-sync`, Cloud Run Job + Cloud Scheduler `* 9-15 * * 1-5` (KST)

**스코프 OUT (의도적 제외):**
- 상한가/하한가/시가총액/거래대금 등 KIS inquirePrice 가 채우는 필드 — 후속 phase

- 키움 dual endpoint 로 KIS 완전 대체 — 후속 phase
- ka10080 분봉차트 backfill — Phase 9 의 EOD 보강만으로 충분

<decisions>
## Implementation Decisions

### API Source
- **D-01:** 키움 REST API `ka10027` (등락률순위) 페이지네이션 사용. 활성 거래 종목만 응답 (~1,898), 매 cycle 평균 ~10 페이지 호출, 합 ~1초.
  - 검증 (2026-05-13): `mrkt_tp="000"` 전체시장, `sort_tp="1"` 상승률 정렬, `cont-yn` + `next-key` 헤더로 페이지네이션. 페이지당 200 행. 10 페이지 = 1,800~1,900 종목.
- **D-02:** `mrkt_tp="000"` (KOSPI+KOSDAQ 통합) 단일 흐름. 시장별 분리 (`mrkt_tp="001"` / `"101"`) 는 사용 X — 통합이 더 효율.
- **D-03:** `updown_incls="1"` (등락 미발생 종목도 응답에 포함) 유지. 거래정지/관리는 응답에서 자연 제외 — 명시적 필터 불필요.
- **D-04:** ⚠️ **거래 없는 종목 ~2,400** 은 본 phase 에서 갱신하지 않음. candle-sync 의 EOD row 가 마지막 close 유지. 의도된 동작 (변화 없으므로 갱신 의미 없음).

### Data Mapping (ka10027 → stock_daily_ohlcv 오늘자 row)
- **D-05:** UPSERT key = `(code, date)`. `code` = `stk_cd` 의 `_AL` 접미사 strip 한 6자. `date` = 오늘 (KST ISO YYYY-MM-DD).
- **D-06:** 매분 갱신 필드:
  - `close` ← `cur_prc` (음수 부호 prefix 의미 = 등락 방향, 절댓값으로 변환)
  - `change_amount` ← `pred_pre`
  - `change_rate` ← `flu_rt`
  - `volume` ← `now_trde_qty` (당일 누적 거래량)
- **D-07:** `open` / `high` / `low` / `trade_amount` 는 **건드리지 않음** — candle-sync EOD 가 채우는 영역. intraday UPSERT 는 `ON CONFLICT (code, date) DO UPDATE SET close=..., change_amount=..., change_rate=..., volume=..., inserted_at=now()` — 명시한 컬럼만 SET.
- **D-08:** **부호 prefix 처리** — `cur_prc="+6760"` / `"-274250"` → Number 변환 시 + 또는 - 가 가격 절댓값에 영향. 등락 방향만 추출 후 가격은 absolute 사용. Plan 단계에서 mapper 함수 명시 (e.g., `parseSignedPrice(s): { price: number, direction: "up"|"down"|"flat" }`).

### Frequency / Scheduler
- **D-09:** Cloud Scheduler cron `* 9-15 * * 1-5` (Asia/Seoul) — 09:00~15:59 매분, 평일. 장 마감 15:30 이후 30분은 잉여이지만 cron 단순성 우선. 빈 응답 / no-op 안전.
- **D-10:** Cloud Run Job task-timeout 60s (충분 — 평균 1초). memory 512Mi. parallelism=1.
- **D-11:** 첫 daily candle-sync (Phase 9, cron `30 17`) 가 오늘자 row 의 **EOD 공식 종가로 덮어씀**. intraday 의 매분 갱신 → EOD 가 최종 확정. 의도된 동작.

### Infrastructure
- **D-12:** **Direct VPC Egress + Static IP** 채택 (Cloud NAT 대체). NAT 라우터 비용 절감 (~$33/월 → 거의 무료).
  - GCP `compute.googleapis.com` API 활성화 + VPC + subnet (asia-northeast3) + Static external IP 1개 reserve + Cloud Run Job 에 `--vpc-egress=all-traffic --network=... --subnet=...`
  - Static IP 는 키움 화이트리스트에 등록 (사용자 액션)
- **D-13:** 새 워커 디렉터리 `workers/intraday-sync/` — master-sync / candle-sync 의 워크스페이스 구조 1:1 mirror.
- **D-14:** GCP IAM SA `gh-radar-intraday-sync-sa` 신규 + `gh-radar-supabase-service-role` secret accessor + scheduler-sa 의 run.invoker.
- **D-15:** 키움 자격증명 GCP Secret Manager 에 저장 (KIWOOM_APPKEY + KIWOOM_SECRETKEY). 신규 secret 2개 + intraday-sync-sa 에 accessor 바인딩.

### Token Management
- **D-16:** 키움 OAuth2 token (24h 유효) 캐시 — Supabase `kiwoom_tokens` 테이블 신규 (kis_tokens 미러 패턴) OR 메모리 내 캐시. 후자 선호 (단일 워커, 매 cycle 재발급 비용 작음). Plan 단계에서 검증.

### Conflict Avoidance
- **D-17:** **테이블 분리** — intraday-sync 는 `stock_daily_ohlcv` 만 건드림. KIS ingestion 의 `stock_quotes` + `top_movers` 는 무변경.
- **D-18:** **시간대 분리** — candle-sync EOD `30 17` 와 intraday `* 9-15` 가 시간대 겹치지 않음. recover `10 8` 도 별도. 동시성 race 없음.
- **D-19:** **API 분리** — KIS (Phase 1 ingestion-worker) 와 키움 (새 intraday-sync) 의 rate-limit bucket 완전 분리. EGW00201 영향 없음.

### Claude's Discretion
- 키움 client 구현 패턴 (axios + interceptors / fetch / SDK) — 키움 docs 권장과 일치하도록
- 부호 prefix 파싱 정확한 정규식
- 에러 분류 (401 / 429 / 5xx) per retry policy 세부 (withRetry 패턴 candle-sync 와 동일)
- 로깅 필드 schema (pino, master-sync 패턴 승계)
- ka10027 응답 변환 유닛 테스트 fixture 구조

<specifics>
## Specific Ideas

- **현재가 부호 prefix 의 의미** (실측): `cur_prc="+6760"` 첫 글자 + = 상승, - = 하락, 빈값 또는 0 = 보합. 등락 방향 마커이며 가격 자체는 절댓값. `pred_pre`, `flu_rt` 도 동일 패턴.
- **stk_cd 의 `_AL` 접미사**: 키움이 응답 시 코드 뒤에 `_AL` 붙임 (예: `007460_AL`). DB 의 `code` 는 6자 단축코드 (예: `007460`). mapper 에서 strip 처리.
- **활성 종목 ~1,898 vs 마스터 ~4,271**: 차이 ~2,373 = 거래정지/관리종목/저활성 ETF/우선주 등. 사용자에게 명시: "거래 발생 종목만 갱신"이 의도.
- **Direct VPC Egress vs Cloud NAT 비교**: Plan 단계에서 GCP docs 재검증. Direct VPC Egress 가 안 되면 Cloud NAT fallback (월 ~$33).

</specifics>
