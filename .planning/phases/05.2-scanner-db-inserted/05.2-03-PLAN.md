    **Step 2 — `server/src/routes/scanner.ts` 수정:**

    (a) `COLS` 상수에 `trade_amount` 추가 (순서는 `volume` 바로 다음):
    ```typescript
    export const COLS =
      "code,name,market,price,change_amount,change_rate,volume,trade_amount,open,high,low,market_cap,upper_limit,lower_limit,updated_at";
    ```
    P-2 pitfall: COLS에 trade_amount 누락 시 Supabase가 해당 필드를 응답에서 생략 → webapp이 undefined 받음. **반드시 추가.**

    (b) route handler 에서 `res.json(rows.map(rowToStock))` **직전**에 `X-Last-Updated-At` 헤더 세팅 블록 추가:

    ```typescript
    // D-10: 헤더 값 = rows 내 MAX(updated_at) ISO 8601 UTC
    // D-11: 0건 시 헤더 생략 (빈 문자열도 금지)
    if (rows.length > 0) {
      const maxMs = Math.max(
        ...rows.map((r) => new Date(r.updated_at).getTime()),
      );
      if (Number.isFinite(maxMs) && maxMs > 0) {
        res.setHeader("X-Last-Updated-At", new Date(maxMs).toISOString());
      }
    }
    res.setHeader("Cache-Control", "no-store");
    res.json(rows.map(rowToStock));
    ```

    구현 메모:
    - `rows` (StockRow[]) 의 `updated_at: string` 은 Supabase가 반환한 ISO 문자열 — `new Date(iso).getTime()` 으로 epoch ms 변환 후 Math.max.
    - `Number.isFinite(maxMs) && maxMs > 0` 방어: 모든 updated_at 이 Invalid Date였을 경우 헤더 생략.
    - 기존 `res.setHeader("Cache-Control", "no-store")` 라인 **유지** — 순서는 X-Last-Updated-At 이후.
    - route handler 위치는 `next(e)` try 블록 **안** — catch 로 error envelope 응답으로 넘어가는 경로에서는 헤더 세팅하지 않음.

    **Step 3 — `server/src/services/cors-config.ts` 수정:**

    `corsOptions()` 반환 객체에 `maxAge: 600,` 바로 아래에 한 줄 추가:
    ```typescript
    exposedHeaders: ["X-Last-Updated-At", "X-Request-Id"],
    ```

    RESEARCH Finding 3 근거:
    - 환경변수 분리 불필요 — 이 헤더 목록은 코드 계약.
    - `X-Request-Id` 도 이미 서버가 세팅 중이라 함께 노출해 일관성 확보.
    - helmet 은 CORS 헤더를 건드리지 않음.
