## DB 스키마 전략

| Option | Description | Selected |
|--------|-------------|----------|
| volume 유지 + trade_amount 신규 추가 (추천) | 둘 다 보존. 향후 유연성 이득, ingestion이 둘 다 upsert. | ✓ |
| volume 제거, trade_amount로 교체 | 불필요한 필드 제거. 단순하지만 재번시블 마이그레이션. | |

## UI 표시

| Option | Description | Selected |
|--------|-------------|----------|
| 거래대금으로 완전 교체 (추천) | volume 컬럼 → trade_amount 컬럼. 가장 직관적. | ✓ |
| 거래량/거래대금 토글 | 사용자가 선택 전환. 복잡도 증가, localStorage 등 추가 설계. | |
| 거래량+거래대금 둘 다 표시 | 정보량 많지만 모바일 가독성 저하. | |

**User's choice:** 거래대금으로 완전 교체
