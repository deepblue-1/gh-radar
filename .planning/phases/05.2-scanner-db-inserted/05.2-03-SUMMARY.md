# Phase 05.2 Plan 03: Server API — trade_amount + X-Last-Updated-At Summary

`/api/scanner` Express 라우트가 (1) `Stock.tradeAmount` 를 응답 body 에 포함하고 (2) `X-Last-Updated-At` 응답 헤더로 DB 기준 최신 갱신시각(ISO 8601 UTC)을 노출하도록 확장. CORS `exposedHeaders` 에 명시해 브라우저가 헤더를 읽을 수 있게 만들었다. body 계약은 `Stock[]` 루트 배열로 불변(envelope 미도입).

## What Was Built

| Layer | File | Change |
|-------|------|--------|
| Mapper | `server/src/mappers/stock.ts` | `StockRow.trade_amount: number` 추가, `rowToStock()` 가 `tradeAmount: r.trade_amount` 반환 |
| Route | `server/src/routes/scanner.ts` | `COLS` 에 `trade_amount` 추가, route 핸들러에 `X-Last-Updated-At` 헤더 세팅 블록 추가 |
| CORS | `server/src/services/cors-config.ts` | `exposedHeaders: ["X-Last-Updated-At", "X-Request-Id"]` |

## Decisions Made

- D-09: body 는 그대로 `Stock[]` — envelope 도입 안 함.
- D-10: 헤더 값 = 응답 rows 의 `MAX(updated_at)` 의 ISO 8601 UTC 문자열.
- D-11: rows 가 0건이거나 모든 `updated_at` 이 invalid 면 헤더 자체를 세팅하지 않음 (빈 문자열 금지).
- D-12: CORS `exposedHeaders` 에 `X-Last-Updated-At` 와 `X-Request-Id` 둘만 화이트리스트 등록.
- D-13: `MAX` 계산은 JS `Math.max(...rows.map((r) => new Date(r.updated_at).getTime()))` 로 단순 처리.
