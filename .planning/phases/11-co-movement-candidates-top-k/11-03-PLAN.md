결합점수·타이트니스·후행 식 (RESEARCH §타이트니스/§두 경로 결합/§후행 — verbatim):
```
theme_weight   = 1 / sqrt(member_count)
// 앵커 참여도 가중 (R3): "X 가 급등한 지금 어느 테마가 발화 중인가"의 프록시 = X 자신의 그 테마 발화 동참률.
// anchor_conf_d0(T) = themeRows 중 stock_code === anchorCode 인 행(테마 찾기에 쓴 앵커 행 — 추가 fetch 불요)의 conf_d0.
// floor 0.2 로 recall 붕괴 방지 (앵커 미동참 테마도 0 이 아닌 ~0.45 배 기여). 메가캡 34테마 노이즈를 member_count 보다 직접 해결.
anchor_rel(T)  = sqrt(ANCHOR_REL_FLOOR + (1 - ANCHOR_REL_FLOOR) · anchor_conf_d0(T))   // ANCHOR_REL_FLOOR = 0.2, 범위 ≈ [0.45, 1]
conf_d0_eff    = max over themes of (conf_d0 · theme_weight · anchor_rel(T))
lift_norm      = min(1, lift / LIFT_CAP)        // LIFT_CAP = 10 (Plan 02 calibration 으로 조정 가능)
avg_ret_norm   = min(1, avg_ret / 30)
combined       = 0.5·conf_d0_eff + 0.2·lift_norm + 0.2·avg_ret_norm + 0.1·conf_d1
strength       = combined  (강도바 width = Math.max(4, Math.min(100, strength*100)))
isTrailing     = (conf_d1 > conf_d0) AND (conf_d1 >= 0.3)
sampleConfidence = ignite_days >= 8 ? "high" : "low"
// dedup: 후보 key = stock_code. evidence 합집합(공유 테마 칩 + coSurgeCount).
//        점수 = max(theme_combined, cosurge_combined). co-surge 전용은 sharedThemes=[], conf_d0=0.
```
