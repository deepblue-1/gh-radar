- 5개 단위 테스트 GREEN

### Task 2: krxToMasterRow + upsertMasters + retry (7c35376)
- `krxToMasterRow`: KRX 실측 응답 → StockMaster 변환
  - C1: sector=NULL (KRX에 업종 정보 없음), kosdaqSegment=SECT_TP_NM 분리
  - C2: securityType=KIND_STKCERT_TP_NM, securityGroup=SECUGRP_NM 별도 보존
  - ISU_ABBRV → name 우선, LIST_DD → ISO date 변환, PARVAL/LIST_SHRS → number
- `upsertMasters`: stocks 테이블 upsert (onConflict:'code') + 9개 신규 메타 컬럼 + dedup by code
- `withRetry`: exponential backoff 200/400/800ms, 3회 시도
- 13개 단위 테스트 GREEN (map 9 + upsert 4)

### Task 3: main 엔트리 파이프라인 + integration test (a8431b9)
- `runMasterSync`: config → KRX fetch → map → upsert → delist-sweep 전체 파이프라인
- MASS_DELIST_RISK 가드: krxRows < 1000 시 throw (부분 응답 대량 오탄 방지)
- delist-sweep: KRX 응답에 없는 활성 종목을 is_delisted=true 마킹
- 빈 응답(0건) 시 count=0 정상 처리 (서비스 승인 미완료 가능성 로그)
- todayBasDdKst: KST 기준 오늘 날짜 자동 계산
- 5개 integration 테스트 GREEN
- typecheck + build 통과, dist/index.js 생성
