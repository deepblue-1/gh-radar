## 2. KRX OpenAPI `stk_isu_base_info` / `ksq_isu_base_info` — **PENDING (AUTH_KEY 미발급)**

**상태:** 사용자가 KRX AUTH_KEY 미보유 → 샘플 호출 불가. Plan 03 `krxToMasterRow` mapper 의 필드명은 RESEARCH Assumption A1(`ISU_SRT_CD/ISU_NM/MKT_TP_NM/SECUGRP_NM/SECT_TP_NM/LIST_DD`) 상태로 남아 있음.

### 2.1 발급 절차 (사용자 수동 — 최대 1일)

**Step 1 — 회원가입**
1. https://openapi.krx.co.kr 접속
2. 우측 상단 **"회원가입"** — 이메일/비밀번호만 요구 (실명인증 없음)
3. 이메일 인증 완료

**Step 2 — AUTH_KEY 발급**
1. 로그인 후 우측 상단 **"마이페이지" → "인증키 관리"**
2. **"인증키 발급"** 버튼 클릭 → 즉시 발급됨 (문자열 1개)
3. **주의:** 키에 공백/개행이 섞이면 401. 복사 시 `trim()` 필수 [RESEARCH Pitfall 5, i-whale.com]

**Step 3 — 서비스 사용 신청 (승인 대기 최대 1일)** ⚠️ 핵심 블로커
1. **"서비스 목록"** 탭에서 다음 2개 서비스 **개별 신청**:
   - **"유가증권 상장종목 기본정보"** (`stk_isu_base_info`)
   - **"코스닥 상장종목 기본정보"** (`ksq_isu_base_info`)
2. 각 서비스 카드 **"신청"** 버튼 → 사유 입력 (예: "개인 투자 정보 조회 앱 개발")
3. **운영진 검토 후 승인** — 최대 1일. 승인 메일 수신 시 사용 가능.

**Step 4 — 샘플 호출 검증 (승인 후)**

```bash
# 어제 영업일(예: 20260414) 기준, KRX는 08:00 KST 이후 당일 데이터 제공
export KRX_AUTH_KEY="발급받은키"

  "https://data-dbg.krx.co.kr/svc/apis/sto/stk_isu_base_info?basDd=20260414" \
  | jq '.OutBlock_1[0]' > /tmp/krx-stk-sample.json

cat /tmp/krx-stk-sample.json /tmp/krx-ksq-sample.json
```
