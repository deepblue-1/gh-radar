## 배포 중 조우한 이슈와 해결

### 이슈 1: Phase 1 Dockerfile `tsconfig.base.json` 누락 (commit 80260e0)
- **증상:** docker build 실패 — `error TS5083: Cannot read file '/app/tsconfig.base.json'`
- **원인:** `extends: "../../tsconfig.base.json"`이 깨져 compilerOptions 전체 미적용 → esModuleInterop/target/downlevelIteration 에러 연쇄
- **해결:** Dockerfile builder stage에 `COPY ... tsconfig.base.json ./` 추가. 동시에 non-root user + `ARG GIT_SHA` / `ENV APP_VERSION` 추가(D-05, D-10 계약 일관성).

### 이슈 2: Phase 1 Dockerfile `Cannot find module 'pino'` 런타임 (commit 27eecfd)
- **증상:** 첫 Job 실행 시 exit code 1 + pino 모듈 ENOENT
- **원인:** pnpm symlink 기반 node_modules를 그대로 COPY하면 심볼릭링크 체인 파괴
- **해결:** `pnpm --filter=@gh-radar/ingestion --prod --legacy deploy /out` 패턴 적용(server/Dockerfile과 동일).

> 두 수정 모두 CONTEXT.md "운영 보강만 허용" 범위 내 — Phase 1이 build까지만 검증하고 실제 container run 테스트를 안 했던 잠재 버그.
