## 실행 중 발견된 Phase 1 잠재 버그 2건 — 즉시 수정

### Bug 1: `tsconfig.base.json` 누락 (commit 80260e0)
- **증상:** docker build 실패. `error TS5083: Cannot read file '/app/tsconfig.base.json'` + esModuleInterop/target/downlevelIteration 에러 연쇄.
- **원인:** Phase 1 Dockerfile이 builder stage에 tsconfig.base.json을 복사하지 않아 workers/ingestion/tsconfig.json의 `extends: "../../tsconfig.base.json"`가 깨지고 compilerOptions 전체 미적용.
- **수정:** `COPY package.json pnpm-workspace.yaml pnpm-lock.yaml tsconfig.base.json ./` — server/Dockerfile 패턴 미러.
- **추가 보강:** non-root user(`app`) + `ARG GIT_SHA` + `ENV APP_VERSION` — CONTEXT.md D-05, D-10 계약 일관성 확보.

### Bug 2: `Cannot find module 'pino'` 런타임 오류 (commit 27eecfd)
- **증상:** 첫 Job 실행 시 `Error: Cannot find module 'pino'` → exit code 1.
- **원인:** pnpm은 symlink 기반 node_modules 구조 → `COPY --from=builder /app/node_modules`로는 심볼릭링크 체인이 깨져 런타임에 모듈을 못 찾음.
- **수정:** `pnpm --filter=@gh-radar/ingestion --prod --legacy deploy /out` 추가 후 `/out/node_modules`를 prod stage로 복사 — server/Dockerfile과 동일 패턴.

두 수정 모두 CONTEXT.md "**제외: 코드 변경(Phase 1 ingestion 로직은 그대로). 환경변수 추가나 종료 코드 정책 등 운영 보강만 허용.**" 범위 내 — 운영 배포 가능한 상태로 만드는 보강.
