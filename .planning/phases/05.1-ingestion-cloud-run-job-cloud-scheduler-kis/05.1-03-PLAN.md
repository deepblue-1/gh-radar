**실행 후 Claude가 자동 검증:**

```bash
REGION=asia-northeast3
JOB=gh-radar-ingestion
PROJECT=$(gcloud config get-value project)

# V1 (VALIDATION Row 05.1-02-01): Job config 정확
gcloud run jobs describe "$JOB" --region="$REGION" --format=json \
  | jq -e '
      .spec.template.spec.template.spec.containers[0].resources.limits.cpu=="1000m"
      and (.spec.template.spec.template.spec.containers[0].resources.limits.memory=="512Mi")
      and (.spec.template.spec.template.spec.timeoutSeconds=="120s")
      and (.spec.template.spec.taskCount==1)
      and (.spec.template.spec.parallelism==1)
      and (.spec.template.spec.template.spec.maxRetries==0)
    ' >/dev/null || { echo "FAIL V1: job config mismatch"; exit 1; }
echo "✓ V1 job config"

# V2 (Row 05.1-02-02): 4개 secret 바인딩
SEC_COUNT=$(gcloud run jobs describe "$JOB" --region="$REGION" --format=json \
  | jq '[.spec.template.spec.template.spec.containers[0].env[] | select(.valueFrom.secretKeyRef!=null).name] | length')
[ "$SEC_COUNT" = "4" ] || { echo "FAIL V2: secret count=$SEC_COUNT"; exit 1; }
echo "✓ V2 4 secrets bound"

# V3 (Row 05.1-02-03): runtime SA 바인딩
SA=$(gcloud run jobs describe "$JOB" --region="$REGION" --format='value(spec.template.spec.template.spec.serviceAccountName)')
[[ "$SA" == "gh-radar-ingestion-sa@${PROJECT}.iam.gserviceaccount.com" ]] \
  || { echo "FAIL V3: runtime SA=$SA"; exit 1; }
echo "✓ V3 runtime SA"

# V4: 이미지 태그 SHA 일치
SHA=$(git rev-parse --short HEAD)
IMAGE=$(gcloud run jobs describe "$JOB" --region="$REGION" --format='value(spec.template.spec.template.spec.containers[0].image)')
echo "$IMAGE" | grep -q "ingestion:${SHA}" || { echo "FAIL V4: image=$IMAGE expected SHA=$SHA"; exit 1; }
echo "✓ V4 image SHA matches"

# V5 (Row 05.1-03-03): scheduler SA가 Job invoker 보유
gcloud run jobs get-iam-policy "$JOB" --region="$REGION" --format=json \
  | jq -e --arg m "serviceAccount:gh-radar-scheduler-sa@${PROJECT}.iam.gserviceaccount.com" \
    '.bindings[] | select(.role=="roles/run.invoker").members | index($m)' >/dev/null \
  || { echo "FAIL V5: scheduler-sa missing invoker on $JOB"; exit 1; }
echo "✓ V5 scheduler invoker bound (resource-scoped)"

# V6 (Row 05.1-03-01): Scheduler OAuth SA
SCHED_SA=$(gcloud scheduler jobs describe gh-radar-ingestion-scheduler --location="$REGION" \
  --format='value(httpTarget.oauthToken.serviceAccountEmail)')
[[ "$SCHED_SA" == "gh-radar-scheduler-sa@${PROJECT}.iam.gserviceaccount.com" ]] \
  || { echo "FAIL V6: scheduler OAuth SA=$SCHED_SA"; exit 1; }
echo "✓ V6 scheduler OAuth SA (not OIDC)"

# V7 (Row 05.1-03-02): Scheduler state/cron/tz
STATE_SCHED_TZ=$(gcloud scheduler jobs describe gh-radar-ingestion-scheduler --location="$REGION" \
  --format='value(state,schedule,timeZone)')
echo "$STATE_SCHED_TZ" | grep -qE '^ENABLED\s+\* 9-15 \* \* 1-5\s+Asia/Seoul$' \
  || { echo "FAIL V7: scheduler state/schedule/tz='$STATE_SCHED_TZ'"; exit 1; }
echo "✓ V7 scheduler ENABLED + cron + Asia/Seoul"

# V8: Scheduler URI = Job :run Google API endpoint (OIDC 오사용 방지 재확인)
URI=$(gcloud scheduler jobs describe gh-radar-ingestion-scheduler --location="$REGION" \
  --format='value(httpTarget.uri)')
[[ "$URI" == "https://${REGION}-run.googleapis.com/apis/run.googleapis.com/v1/namespaces/${PROJECT}/jobs/${JOB}:run" ]] \
  || { echo "FAIL V8: URI=$URI"; exit 1; }
echo "✓ V8 scheduler URI correct"

# V9: 이미지가 Artifact Registry에 실제 존재
gcloud artifacts docker images describe \
  "asia-northeast3-docker.pkg.dev/${PROJECT}/gh-radar/ingestion:${SHA}" >/dev/null \
  || { echo "FAIL V9: image not in AR"; exit 1; }
echo "✓ V9 image in AR"

echo ""
echo "✅ Wave 2 complete — Job + Scheduler live. Wave 3 next (alert policy)."
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 사용자 로컬 → Artifact Registry | Docker image layers 전송. `gcloud auth configure-docker` 설정된 helper가 OAuth 토큰으로 자동 인증. |
| Cloud Scheduler → Cloud Run Job `:run` API | Google API 엔드포인트. OAuth 액세스 토큰(`gh-radar-scheduler-sa`) 사용 — OIDC **금지**. |
| Cloud Run Job runtime → Secret Manager | 런타임 시 Secret Manager API가 `gh-radar-ingestion-sa` IAM 확인 후 값 주입. |
| Cloud Run Job runtime → KIS OpenAPI / Supabase | TLS(HTTPS). 자격증명은 env로 주입된 secret 값 사용. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.1-01 | Information Disclosure | `gcloud run jobs deploy` 커맨드라인 | mitigate | 모든 credential은 `--set-secrets=SECRET_ID:latest` 참조만 사용 (`--set-env-vars` 평문 주입 금지). SUPABASE_URL은 public endpoint이므로 평문 env OK. |
| T-05.1-02 | Elevation of Privilege | Section 5.5 IAM binding | mitigate | `gcloud run jobs add-iam-policy-binding <JOB>` **리소스 단위** (프로젝트 단위 `gcloud projects add-iam-policy-binding ... roles/run.invoker` 금지). |
| T-05.1-03 | Spoofing | Scheduler → Job `:run` | mitigate | `--oauth-service-account-email=gh-radar-scheduler-sa@...` 강제 (RESEARCH Pitfall 2). URI는 `run.googleapis.com` Google API 전용. V6/V8 검증에서 `oidcToken` 부재 명시 확인. |
| T-05.1-05 | Tampering | `:latest` 태그 | mitigate | 배포마다 `:SHA`를 명시적으로 Job `image` 필드에 바인딩(V4). `:latest`는 편의용 포인터로만 사용. Cloud Run이 `:SHA` 고정으로 revision 관리. |
| T-05.1-08 | Denial of Service | Scheduler overlap (Pitfall 1) | accept | 실측 관찰. upsert 멱등성(Phase 1 D-15)으로 데이터 오염 없음. 추후 overlap 빈번하면 Scheduler pause/cron 조정 — Deferred. |
| T-05.1-09 | Information Disclosure | Cloud Run metadata (env) | mitigate | `--set-secrets` 사용 시 Cloud Run metadata에는 secret reference만 저장(값 X). `gcloud run jobs describe`의 env list에서 `valueFrom.secretKeyRef`만 노출됨 — V2가 이를 확인. |
</threat_model>
