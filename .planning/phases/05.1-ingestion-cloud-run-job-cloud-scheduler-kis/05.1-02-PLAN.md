**실행 후 검증 명령 (Claude가 실행 가능):**

```bash
# V1: API 6종 enabled (VALIDATION Row 05.1-01-02 연계)
for API in run.googleapis.com cloudscheduler.googleapis.com secretmanager.googleapis.com \
           monitoring.googleapis.com artifactregistry.googleapis.com iam.googleapis.com; do
  gcloud services list --enabled --filter="config.name=$API" --format='value(config.name)' | grep -q "$API" \
    || { echo "FAIL: $API not enabled"; exit 1; }
done
echo "✓ V1 APIs enabled"

# V2: SA 2종 존재 (VALIDATION Row 05.1-01-03)
SA_COUNT=$(gcloud iam service-accounts list \
  --filter='email ~ gh-radar-(scheduler|ingestion)-sa@' \
  --format='value(email)' | wc -l | tr -d ' ')
[ "$SA_COUNT" = "2" ] || { echo "FAIL: SA count=$SA_COUNT, expected 2"; exit 1; }
echo "✓ V2 service accounts (2)"

# V3: Secret 4종 존재 (VALIDATION Row 05.1-01-02)
SECRET_COUNT=$(gcloud secrets list \
  --filter='name ~ gh-radar-(kis-app-key|kis-app-secret|kis-account-number|supabase-service-role)$' \
  --format='value(name)' | wc -l | tr -d ' ')
[ "$SECRET_COUNT" = "4" ] || { echo "FAIL: secret count=$SECRET_COUNT, expected 4"; exit 1; }
echo "✓ V3 secrets (4)"

# V4: 각 secret에 :latest 버전 존재
for S in gh-radar-kis-app-key gh-radar-kis-app-secret gh-radar-kis-account-number gh-radar-supabase-service-role; do
  gcloud secrets versions describe latest --secret="$S" --format='value(state)' | grep -q ENABLED \
    || { echo "FAIL: $S latest not ENABLED"; exit 1; }
done
echo "✓ V4 all secrets have :latest enabled"

# V5: Runtime SA가 모든 secret에 secretAccessor 바인딩 보유
PROJECT=$(gcloud config get-value project)
for S in gh-radar-kis-app-key gh-radar-kis-app-secret gh-radar-kis-account-number gh-radar-supabase-service-role; do
  gcloud secrets get-iam-policy "$S" --format=json \
    | jq -e --arg member "serviceAccount:gh-radar-ingestion-sa@${PROJECT}.iam.gserviceaccount.com" \
        '.bindings[] | select(.role=="roles/secretmanager.secretAccessor").members | index($member)' >/dev/null \
    || { echo "FAIL: $S missing accessor for gh-radar-ingestion-sa"; exit 1; }
done
echo "✓ V5 secretAccessor bound to gh-radar-ingestion-sa on 4 secrets"

echo ""
echo "✅ Wave 1 complete — ready for Wave 2 (deploy-ingestion.sh)"
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 사용자 셸(로컬) → GCP API | KIS/Supabase credential이 사용자 셸 env에서 gcloud API로 전송. HTTPS만 허용. |
| GCP API → Secret Manager (Google KMS) | Secret Manager가 envelope encryption으로 저장. 사용자는 후속 단계에서 IAM 바인딩으로만 접근. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.1-01 | Information Disclosure | 셸 env → stdin 전송 경로 | mitigate | `printf '%s' "$value" \| gcloud secrets versions add ... --data-file=-` stdin 주입. 커맨드라인 인자 금지. `read -rs`로 terminal echo 차단. history는 앞 공백 prefix로 비저장. |
| T-05.1-02 | Elevation of Privilege | SA 생성/바인딩 | mitigate | Runtime SA에는 **오직** `roles/secretmanager.secretAccessor`만 부여. 프로젝트 단위 바인딩 금지 — secret 리소스 단위 바인딩만. |
| T-05.1-04 | Information Disclosure | Secret Manager version 저장 | accept | Google KMS envelope encryption 의존. 하드코딩은 안 함. |
| T-05.1-07 | Repudiation | Secret 생성 감사 | accept | Cloud Audit Logging이 자동으로 `AddSecretVersion` 등 기록. 별도 조치 불필요. |
</threat_model>
