# 새 세션용 프롬프트 — 상한가 동조 종목 탐지 (아이디어 디스커션)

> 아래 블록을 그대로 새 세션에 붙여넣으면 됨. Phase 10 직전 아이디어 회의(세션 `2286945e`)에서
> 논의·리서치까지 끝났으나 "Phase 10 다음 별도 phase"로 분리된 뒤 로드맵에 추가 안 된 아이디어를
> 이어서 진행하기 위한 출발점.

---

gh-radar 프로젝트에서 아직 구현 안 된 아이디어 하나를 이어서 논의하고 싶어. **바로 구현하지 말고, 먼저 아이디어 디스커션 + 리서치**부터 하자.

## 배경
이 프로젝트는 한국 주식 "상한가 따라잡기" 정보 제공 앱이야. Phase 10(테마 분류)을 추가하기 전에 아이디어 회의를 했는데, 그때 **두 가지** 아이디어를 냈었어:
1. 테마별 종목 묶기("이재명주" 같은 테마 → 소속 종목 리스트) → **Phase 10으로 구현 완료**.
2. **특정 종목이 상한가를 갈 때, 같이 상한가를 갈 만한 종목을 찾기** — 그동안의 일봉 차트로 상관관계를 수치화해서 찾아내기. ← 이게 "Phase 10 다음의 별도 phase로 후속"으로 분리됐는데, **그 후속 phase가 아직 안 만들어졌어.** 이번에 이걸 진행하고 싶어.

## 핵심 목표
종목 X가 방금 상한가/급등했을 때 → **"X를 따라 오를 만한 후보 종목 Y들"** 을 과거 데이터 기반으로 점수화해 보여주기. (테마와는 다른 축의 신호 — 통계적 동조/선행.)

## 이미 끝난 리서치 (다시 처음부터 하지 말고 출발점으로 삼되, 현재 코드베이스로 재검증)

**가용 인프라 (재사용 가능 — 신규 데이터 수집 부담 작음):**
- `stock_daily_ohlcv` — 일봉 OHLCV, 2020-01~현재 4M+ 행 (`workers/candle-sync`). → 동조 분석 즉시 가능.
- `stocks` 마스터 ~2,800종목 · `stock_quotes` 인트라데이(현재가/등락률) · **`themes`/`theme_stocks`(Phase 10 산출 — 테마 게이팅에 활용 가능)**.
- 신규로 필요한 것: **동조 페어 점수 배치 워커** + **점수 저장 테이블** + **종목 상세 페이지 UI(동조 후보)**.

**알고리즘 옵션 (당시 정리):**
| 옵션 | 방식 | 평가 |
|------|------|------|
| A. Pearson 상관계수 | 최근 60~120일 일별 수익률(%) 상관 | 단순/표준이나 "평상시 동조"라 급등 시나리오엔 약함 |
| **B. 상한가 이벤트 Co-movement** | A가 상한가 친 날들에 B가 동반 급등(+X%↑)한 빈도/평균 | **우리 유즈케이스에 직접 부합 — 추천** (단, 이벤트 희소 종목은 데이터 부족) |
| **C. 테마 내 상관 결합** | 같은 테마로 후보 풀을 좁혀 노이즈↓ (Phase 10 themes 활용) | **추천(B와 결합)** — 전종목 N² 회피 + 설명 쉬움 |
| D. Lead-lag / Granger | A가 B를 며칠 선행하는지 | "선행 종목" 찾기 강력하나 복잡 → v2 |
| E. Cosine similarity | 수익률 벡터 cos sim | 빠르나 A와 유사 |

**당시 권장: v1 = B(상한가 이벤트 동조) + C(테마 게이팅). A는 보조, D/E는 v2.**

**v1 알고리즘 초안:**
```
입력: 종목 X (방금 상한가)
1) X의 테마 T 조회 (Phase 10 theme_stocks)
2) T 소속 다른 종목 Y 후보 추출 (전종목 N² 회피)
3) 과거 N개월(예: 12개월) X가 상한가/급등(+15%↑) 친 날들 → 이벤트 집합 E
4) 각 Y에 대해 E의 날들에서 Y 수익률 평균/중앙값 + (+10%↑) 빈도 계산
5) 상위 K개를 "동조 후보"로 점수화해 반환
```

**B를 고른 근거(한국 시장 특성):** ±30% 가격제한폭 · 개인 비중 60%+ · 테마 3~5일 회전 · "평상시 상관 0.3 → 테마 발화 시 0.9로 점프" 패턴 → 평균 상관(A)보다 이벤트 조건부 동조(B)가 직접적.

## 같이 정하고 싶은 것 (디스커션 안건)
1. **알고리즘**: B+C 확정인가, 다른 조합인가? 이벤트 희소(상한가 드문 종목) 보완책은? 테마가 없는 종목은 어떻게 후보를 잡나(섹터/전종목 fallback)?
2. **점수 정의**: 동반급등 빈도 vs 평균수익률 vs 결합 스코어? lookback 기간? 동일일 동조 vs 선행/후행(다음날) 구분?
3. **데이터/성능**: 전종목 페어 사전계산(배치) vs 후보만 on-demand? 점수 테이블 스키마/갱신 주기? 4M+ 행에서 페어 계산 비용?
4. **UI**: 종목 상세 페이지에 "동조 후보 TOP K" 섹션? 신뢰도·근거(같이 오른 날 수 등) 표시 방식?
5. **스코프/순서**: v1 최소 범위는 어디까지? 별도 phase로 `/gsd-add-phase` 진행할지.

먼저 코드베이스 현황(`workers/candle-sync`, `stock_daily_ohlcv` 스키마, `themes`/`theme_stocks`, 스캐너/종목상세 UI)을 확인하고, 위 안건들을 리서치·검토해서 같이 정하자.

**운영 규칙:** 한글로 소통 · GSD 워크플로우(비자명 작업은 plan mode) · 커밋은 한글 메시지 + 사전 확인 + Co-Authored-By 없이.
